Normative contract for the tabletop skirmish initiative audit. All inputs are UTF-8 JSON with ASCII-only strings. Every output file is UTF-8 JSON, ASCII-only, two-space indentation, recursively sorted object keys at every depth, no trailing spaces at line ends, and exactly one trailing newline after the root closing brace.

Read `policy.json` for `current_day`, `stance_delta` (keys `attack`, `defend`, `neutral`), `faction_favor` (integer bonuses, missing factions default to zero), `band_rules` (objects with integer `min_priority` and integer `band`), integer `stim_delta` (default zero when absent), integer `brace_lift` (treat as two when the field is absent or not a positive integer), and optional integer `echo_max_abs_priority_delta` (omit the field entirely to disable the extra echo coupling check). Read `incidents.json` for `events` (objects with integer `day`, string `kind`, string `unit_id`). Read `overclock.json` for `units` (array of unit ids). Enumerate every `*.json` file directly under `units/` (ignore subdirectories) and parse each object, which must contain string `unit_id`, integer `base_priority`, string `stance`, and string `faction`. An optional string `escort` names another unit id; when present it must reference a different observed unit. For each `unit_id`, read `loadouts/<unit_id>.json` containing integer `tie_token`.

For each unit, scan `events` restricted to `day` equal to `current_day` and matching `unit_id`. Recognize these `kind` strings: `stim`, `thermal_throttle`, `jam`, `brace`, and `suppress`. Other kinds are ignored. Let `has_stim` be true when any matching event has `kind` exactly `stim`. Let `suppress_stim` be true when any matching event has `kind` exactly `suppress`. Let `braced` be true when any matching event has `kind` exactly `brace`. Let `throttle_hits` be the count of matching events whose `kind` is exactly `thermal_throttle`.

Define `base_priority_score` as `base_priority` plus `stance_delta[stance]` (missing stance key falls back to the integer stored under `neutral`, and a missing `neutral` key contributes zero) plus `faction_favor[faction]` when present, plus `brace_lift` from `policy.json` (using the positive-default rule above) whenever `braced` is true. Define `stimmed` true when `has_stim` is true and `suppress_stim` is false. Define `priority_score` as `base_priority_score` plus `stim_delta` when `stimmed` is true, otherwise equal to `base_priority_score`. Define `intrinsic_band` by sorting `band_rules` by `min_priority` descending and choosing the `band` field from the first rule in that order for which `priority_score` is greater than or equal to that rule's `min_priority`.

Define `throttled` true when `throttle_hits` is strictly greater than zero. Define `jammed` true when any matching event has `kind` exactly `jam`. Jam events on other days do not set `jammed`. Define `overclock_eligible` true when `unit_id` appears in `overclock.json` `units`.

Define `final_band` as an integer. If `throttled` is true, set `final_band` to `intrinsic_band` plus `throttle_hits`, then clamp to the maximum `band` value present in `band_rules` (inclusive ceiling). If `throttled` is false and `overclock_eligible` is true, set `final_band` to `intrinsic_band` minus one, then clamp to zero as a floor. Otherwise set `final_band` to `intrinsic_band`. Throttling suppresses overclock movement even when both would otherwise apply.

After every unit has an initial `final_band`, apply escort anchoring. For each unit U with an `escort` field naming unit E, whenever `final_band` of U is strictly less than `final_band` of E, raise U's `final_band` to match E. Repeat this scan over all escort edges until a full pass makes no change. If escort edges form a directed cycle, every unit in that cycle must end with the maximum `final_band` among cycle members after the iterative pass completes.

Let `roster_max_final_band` be the maximum `final_band` observed on any unit after escort anchoring. Define `echo_sunk` true for unit Y when Y is not `jammed` and there exists a different unit X such that X is `jammed`, X's `faction` equals Y's `faction`, and X's `intrinsic_band` equals Y's `intrinsic_band`. When `echo_max_abs_priority_delta` is present in `policy.json`, require additionally that the absolute difference between X's `priority_score` and Y's `priority_score` is less than or equal to that integer; when the field is absent, omit this extra inequality.

Build the clash ledger before turn reordering. For every unordered pair of distinct units with identical `base_priority_score` and identical `intrinsic_band`, emit one object with keys `intrinsic_band`, `members`, `shared_priority_score`. Omit the pair when either member is `throttled`. `members` must be the two ids sorted ascending. Sort the clash objects ascending by `members[0]`, then `members[1]`, then `shared_priority_score`, then `intrinsic_band`.

Sort units using comparator order: lower `final_band` first; when `final_band` ties and equals `roster_max_final_band`, lower `priority_score` first; when `final_band` ties and is not `roster_max_final_band`, higher `priority_score` first; when those tie, ascending `unit_id`; when those tie, ascending `tie_token`. After this primary sort, perform a stable partition within each contiguous block sharing the same `final_band`: every unit with neither `jammed` nor `echo_sunk` first; then every unit with `echo_sunk` true and `jammed` false; then every unit with `jammed` true, preserving relative order inside each subgroup.

Emit `turn_order.json` with keys `ordered_unit_ids` and `per_unit`. `ordered_unit_ids` lists every observed `unit_id` in final order. `per_unit` maps each id to an object with boolean `echo_sunk`, boolean `jammed`, boolean `overclocked` (true only when `overclock_eligible` is true, `throttled` is false, and `final_band` is not equal to `intrinsic_band`), boolean `throttled`, integer `final_band`, integer `intrinsic_band`, integer `priority_score`.

Emit `clashes.json` with key `clashes` holding the sorted clash objects described above.

Emit `summary.json` with integer `clash_count` (length of `clashes`), integer `current_day` copied from policy, integer `echo_sink_count` (count of units with `echo_sunk` true), object `band_counts` mapping each decimal string `final_band` value observed on any unit to its count (omit bands with zero counts), integer `total_units`, and object `status_counts` with non-negative integers for `delayed`, `degraded`, and `ready`. A unit with `throttled` true counts toward `degraded` even when `jammed` is also true. Otherwise, a unit with `jammed` true counts toward `delayed`. All remaining units count toward `ready`. `echo_sunk` alone does not change status bucketing.

Canonical numeric encoding writes integers without leading zeros or plus signs. Booleans are lowercase tokens `true` and `false`.
