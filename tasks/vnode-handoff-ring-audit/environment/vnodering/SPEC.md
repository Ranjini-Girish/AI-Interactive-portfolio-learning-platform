Normative contract for the vnode handoff ring audit. Inputs are UTF-8 JSON with ASCII-only strings. Outputs are UTF-8 JSON, ASCII-only, two-space indentation, recursively sorted object keys at every depth, no trailing spaces at line ends, and exactly one trailing newline after each root closing brace.

Read `policy.json` for float `vote_ratio` between zero and one inclusive, integer `warmup_moves` (non-negative), integer `ring_modulus` (positive), and float `spill_penalty` (non-negative). Read `manifest.json` for `shard_tag` and `active_shard`. When they differ, add `spill_penalty` to `vote_ratio` capped at `1.0`; round `effective_vote_ratio` in summary to six decimals. Read `epochs.json` for integer `current_epoch`. Read `keys.json` for array `keys` with string `key_id` and integer `hash_slot`. Read `moves.json` for array `moves` with integer `step`, string `key_id`, and string `target_vnode`.

Enumerate every `*.json` under `vnodes/`. Each vnode has string `vnode_id`, integer `range_start`, integer `range_end`, float `weight`, and integer `epoch`. A vnode is stale when `epoch` is strictly less than `current_epoch - 1`. Packaging under `anchors/`, `ancillary/`, `meta/`, and `grid/` is ignored.

Initial key ownership: for each key, `hash_slot` modulo `ring_modulus` falls in a vnode range when `range_start <= slot <= range_end`, or when `range_start > range_end` and slot is at least `range_start` or at most `range_end`. Among active vnodes covering the slot, the owner is the vnode with strictly greatest `weight`; ties keep the first vnode in ascending `vnode_id` among tied weights.

Process moves in ascending `step`, then `key_id`. Keys absent from initial ownership emit `orphan_key`. Warmup and stale-target rows emit `warmup_skipped` or `stale_skipped`. Handoff vote at a step: `agree_weight` sums `weight` of active vnodes appearing as `target_vnode` on the same `step`. The move is accepted when `agree_weight >= vote_ratio * total_active_weight` where `total_active_weight` sums active vnode weights once. When rejected, emit `handoff_rejected`. When accepted and `target_vnode` differs from current owner, record a handoff and update ownership; same owner yields `noop_handoff`.

Emit `move_outcomes.json` with `moves` in processing order (current_owner, key_id, outcome, step, target_vnode). Emit `handoff_plan.json` with `handoffs` sorted by step then key_id (from_vnode, key_id, step, to_vnode). Emit `ring_violations.json` with `violations` sorted by step then key_id. Emit `vnode_states.json` with `vnodes` sorted by vnode_id. Emit `summary.json` with totals and effective_vote_ratio.

Read `VHR_DATA_DIR` defaulting to `/app/vnodering` and `VHR_AUDIT_DIR` defaulting to `/app/audit`. Create the audit directory when missing and never mutate inputs.
