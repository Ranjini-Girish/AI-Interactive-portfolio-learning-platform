Implement a power spectral density estimation tool in TypeScript on Node 22 that reads time-series signals from `/app/data/signals/` and analysis configuration from `/app/data/config.json`, then writes a complete spectral report to `/app/output/spectral_report.json`. Compiled binaries go in `/app/build/`. The environment provides `the TypeScript toolchain` and `python3`. Do not modify any file under `/app/data/`.

Each signal file contains a JSON object with signal_id, sample_rate_hz, and a samples array. The config specifies default and per-signal window functions, Welch parameters, peak detection thresholds, and output rounding precision. The DFT computation, PSD normalization, one-sided spectrum construction, window function definitions, Welch averaging procedure, peak prominence algorithm, SNR formula, and the complete output schema are documented in the `.md` files under `/app/docs/`. The tool must conform to every specification described there.

The output JSON uses 2-space indentation with a trailing newline. The report contains schema_version (integer 1), total_signals_analyzed, analysis_config, a signals array with per-signal results, and a summary with aggregate counts.
