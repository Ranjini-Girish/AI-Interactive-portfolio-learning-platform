Normative contract for the DNS TTL floor audit. Inputs are UTF-8 JSON with ASCII-only strings. Outputs are UTF-8 JSON, ASCII-only, two-space indentation, recursively sorted object keys at every depth, no trailing spaces at line ends, and exactly one trailing newline after each root closing brace.

Read `policy.json` for integer `ttl_floor` (positive), integer `warmup_queries` (non-negative), integer `cname_depth_max` (positive), and float `spill_ratio` (positive). Read `manifest.json` for `zone_tag` and `active_zone`. When they differ, effective ttl floor is `max(1, round(ttl_floor * spill_ratio))`; round `effective_ttl_floor` in summary to an integer. Read `epochs.json` for integer `current_epoch`. Read `queries.json` for array `queries` with integer `step` and string `record_id`.

Enumerate every `*.json` under `records/`. Each record has string `record_id`, string `rtype` in `A` or `CNAME`, integer `ttl`, integer `epoch`, and optional string `target` for CNAME rows. A record is stale when `epoch` is strictly less than `current_epoch - 1`. Packaging under `anchors/`, `ancillary/`, `meta/`, and `grid/` is ignored.

Process queries in ascending `step`, then ascending `record_id`. For missing record ids emit `missing_record`. When `step` is less than or equal to `warmup_queries`, emit `warmup_skipped` without hop or violation rows. When stale, emit `stale_skipped`. Otherwise walk CNAME chains starting at the queried record: append each visited `record_id`, apply per-hop ttl `max(effective_ttl_floor, record.ttl)`, follow `target` on CNAME rows until an `A` row ends the chain. When follow depth exceeds `cname_depth_max`, emit `cname_depth_exceeded`. When a CNAME `target` is missing, emit `broken_chain`. When the chain completes on an `A` row, `effective_ttl` is the minimum per-hop ttl along the chain. When the queried record's raw `ttl` is strictly less than effective ttl floor, emit `floor_breach` with violation row; otherwise outcome is `ok` and emit a `cname_hops` row with the chain and effective ttl.

Emit `query_outcomes.json` with `queries` in processing order (effective_ttl, outcome, record_id, stale, step). Emit `cname_hops.json` with `hops` sorted by step then record_id for ok outcomes only. Emit `floor_violations.json` with `violations` sorted by step then record_id. Emit `record_states.json` with `records` sorted by record_id. Emit `summary.json` with totals and effective_ttl_floor.

Read `DTF_DATA_DIR` defaulting to `/app/dnsfloor` and `DTF_AUDIT_DIR` defaulting to `/app/audit`. Create the audit directory when missing and never mutate inputs.
