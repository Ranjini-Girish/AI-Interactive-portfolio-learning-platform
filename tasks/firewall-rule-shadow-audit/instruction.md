Implement a deterministic stateless-firewall auditor. Read three JSON inputs from `/app/data/` — `rules.json` (`{"rules":[…]}`), `flows.json` (`{"flows":[…]}`), and `policy.json` with keys `default_action` ∈ {"allow","deny"}, `enable_directionality` (bool), and `tie_breaker` ∈ {"priority_then_id","priority_lowest_wins","id_only"} — and write six canonical JSON reports to `/app/output/`: `flow_verdicts.json`, `rule_analysis.json`, `policy_summary.json`, `equivalence_classes.json`, `rule_dependencies.json`, and `perturbation_graph.json`. Do not modify anything under `/app/data/`.

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/replay` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

Every rule has `id`, `priority` (any signed int), `action` ("allow"/"deny"), `direction` ("ingress"/"egress"), `source` and `destination` (the literal "any" or an IPv4 CIDR — canonicalize by masking host bits), `protocol` ("tcp"/"udp"/"icmp"/"any"), `src_ports` and `dst_ports` (the literal "any" or a list of inclusive `{"start":int,"end":int}` ranges), and an advisory `description` that must not influence any output. A rule **matches** a flow iff its predicate is satisfiable, the directionality constraint holds when `enable_directionality` is true, source/destination CIDRs contain the flow IPs ("any" matches everything), the protocol matches ("any" matches everything), and — for tcp/udp flows only — both ports fall inside some configured range (icmp and other non-tcp/udp flows ignore port specs entirely). Predicates are **unsatisfiable** when source/destination is neither "any" nor a valid IPv4 CIDR (IPv6 and masks like /33 are invalid), or when a port spec is not "any" and contains no integer range with start ≤ end. The global **evaluation order** is built from `tie_breaker`: `priority_then_id` puts higher priority first, `priority_lowest_wins` puts lower priority first, `id_only` ignores priority; ties within priority are always broken by ASCII-ascending `id`. Flows are evaluated first-match-wins against this order; the matching rule's action is the flow's `verdict`, or the literal string `"default"` with `matched_rule_id: null` when no rule matches.

Every output file must be UTF-8, two-space-indented, ASCII-only (`ensure_ascii=True`), with keys lexicographically sorted at every depth and a single trailing newline. `flow_verdicts.json` carries `flows` (sorted by id) of `{evaluated_rule_ids, id, matched_rule_id, verdict}`, where `evaluated_rule_ids` is the contiguous prefix of the evaluation order actually inspected (including the matching rule, or the entire order when nothing matched). `rule_analysis.json` carries `rules` (sorted by id) of `{coverage_percent, id, matched_flows, shadowed_by, status}`, where `matched_flows` is the sorted list of flow ids the rule's predicate matches (priority and ordering are ignored — pure predicate match), `coverage_percent` is `len(matched_flows)/len(flows)*100` rendered as a string with **exactly two fractional digits using banker's rounding** (`"0.00"` when there are no flows), `status` is the first-applicable of (a) `unreachable` if `matched_flows` is empty, (b) `shadowed` if every matched flow is also matched by some rule **strictly earlier** in the evaluation order, (c) `redundant` if removing the rule does not flip any flow's allow-vs-deny outcome with `"default"` normalized to `policy.default_action`, otherwise (d) `effective`; `shadowed_by` is non-empty iff `status == "shadowed"` and is the **smallest-cardinality** subset of those earlier rule ids whose predicate-match sets together cover this rule's `matched_flows`, with the lexicographically smallest sorted-id list breaking ties of equal cardinality. `policy_summary.json` carries the four `status` counts plus `total_rules`, `default_action_uses` (count of `verdict=="default"` flows), and `escalation_warnings`: one `{earlier_rule_id, rule_id, type:"deny_after_allow"}` entry per ordered (allow-earlier, deny-later) pair whose predicate-match sets share at least one flow, **only when the deny rule's status is neither `unreachable` nor `redundant`** (a `shadowed` deny **is** included), sorted by `(rule_id, earlier_rule_id)`. `equivalence_classes.json` carries `{minimal_rule_ids, removed_rule_ids, verdict_invariant:true}` produced by deterministic greedy elimination: at each step consider candidates in ASCII-ascending id order, accept the **first** removal that still preserves the `(verdict, matched_rule_id)` pair for **every** flow against the reduced set under the same `tie_breaker`, and restart the scan after every accepted removal until none can be dropped. `rule_dependencies.json` carries one entry per input rule (sorted by id) with exactly five keys `{criticality, flows_changed, id, promoted_rules, verdict_changes}` derived from a one-rule-removed perturbation: `flows_changed` is the sorted list of flow ids whose `(verdict, matched_rule_id)` pair changed under the perturbed rule set; `verdict_changes` is the sorted-by-flow-id list of `{flow_id, from_verdict, to_verdict}` for flows whose **normalized** verdict actually flipped (with `"default"` mapped to `policy.default_action` on both sides — never emit the literal `"default"` here); `promoted_rules` is the sorted list of other rule ids whose perturbed status became `effective` and whose **baseline** status was not `effective`; `criticality` is the first of `"critical"` (verdict_changes non-empty) → `"important"` (flows_changed non-empty) → `"minor"` (promoted_rules non-empty) → `"none"`. `perturbation_graph.json` carries exactly four keys `{cycles, edges, nodes, topological_layers}` over the directed graph induced by `rule_dependencies` (one node per rule; edge `R → Q` iff `Q ∈ R.promoted_rules`; no self-loops; collapse multi-edges): `edges` sorted by `(from, to)`; `nodes` sorted by id with `{id, in_degree, out_degree}` from the deduplicated edge set; `topological_layers` are the Kahn-style layers of the SCC condensation flattened to ASCII-sorted rule ids per layer in layer-index order; `cycles` lists every multi-vertex SCC as its sorted member ids, with the outer list sorted by each cycle's lex-smallest member. The verifier recomputes a reference from `/app/data/` and compares for exact structural equality, so do not hardcode answers.
