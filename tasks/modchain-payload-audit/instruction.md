Synthetic lab payloads under `/app/mod_chain_lab/` describe framed byte blobs, stream membership, incident windows, and a pool cap together with the binding `SPEC.md`. The spec defines lowercase-hex parsing, tier biases, ordered modular mixing, which incident kinds delete frames or add per-stream bias during an active day window, when a stream is treated as fully quarantined after an accepted compromise event, and how pooled caps rescale reported terminals when non-quarantined raw totals exceed the cap. Nothing outside the catalog’s referenced frame files participates; leave every other path under the lab directory untouched.

Build a Rust Cargo project rooted at `/app` using the toolchain already in the image, compile with `cargo build --release`, and place the executable at `/app/target/release/modchain`. Running that binary with no arguments must read only the lab directory contents, emit exactly one regular file `/app/audit/mod_digest.json`, and exit zero on the bundled fixtures. The report must be UTF-8 JSON with two-space indentation, ASCII-only string values, every object key sorted lexicographically at all nesting levels, and exactly one trailing newline at end-of-file; two successful runs must produce byte-identical output.
