A read-only module graph lives at the absolute path named by GMBA_MODGRAPH_DIR, defaulting to /app/modgraph/, with workspace manifest, modules/, registry/, advisories/, incident log, and SPEC.md. Read SPEC.md first for input grammar, version order, five-phase pipeline, sort orders, and JSON encoding (two-space indent, sorted keys, one trailing newline, no HTML escaping).

The agent writes Go under /app/src/, builds /app/bin/modarb, and runs it. The binary reads modgraph data only from that resolved root and writes all five JSON files into the directory named by GMBA_DECISIONS_DIR, defaulting to /app/decisions/. When a variable is set, its path overrides the default; every artifact must appear under the resolved decisions directory even when /app/decisions/ is missing. It must compute outputs only from the modgraph; it must not read existing decision files as sources. Load all of SPEC.md in full; partial reads drop required rules. The five outputs are version_resolution.json (seven actions), replace_directive_audit.json (five statuses), vulnerability_exposure.json (six statuses), cycle_and_retract_report.json, and summary.json (fourteen integer counters). Do not modify the modgraph tree the binary reads. Paths are absolute Linux paths in the container.
