Platform operations needs a deterministic same-day patch-slot plan for every fleet host after bundle dependency rules, tier-and-host maintenance windows, per-region scheduling caps with incident-driven capacity bumps, reboot-minute budgets, bundle embargoes, host compromise holds, and region freeze directives are all evaluated against `pool_state.current_day`. The result must be replayable without touching the frozen input tree.

Write five UTF-8 JSON artifacts under `/app/audit/` named `host_plan.json`, `region_ledger.json`, `bundle_matrix.json`, `incident_journal.json`, and `summary.json`. Grammar for hosts, bundles, regions, incidents, blocked-candidate reasons, host status values, and canonical JSON formatting lives in `/app/patch_slots/SPEC.md`.

Inputs are under `/app/patch_slots/`. When `PSLA_DATA_DIR` is set to a non-empty value it overrides the input root; when `PSLA_AUDIT_DIR` is set to a non-empty value it overrides the output directory. Unset or empty variables fall back to `/app/patch_slots/` for reads and `/app/audit/` for writes. Create the output directory if needed. Do not modify files under the input root during the audit run.
