Reliability needs a frozen storefront edge probe readout that merges HTTP-style status classing, RTT SLO buckets, integer throughput from byte counts, per-region median absolute deviation outlier flags, ancillary minimum-throughput demotions, and incident hints drawn from both the incident log and anchor text lines without mutating raw measurements.

Emit five UTF-8 JSON files under `/app/audit/` named `endpoint_profiles.json`, `tier_rollups.json`, `anomaly_events.json`, `mad_summary.json`, and `summary.json`. Canonical JSON rules, tier vocabulary, per-region MAD math, throughput integer division, demotion order, ancillary merges, anchor parsing, and incident override semantics are all defined in `/app/edgeprobes/SPEC.md`. Inputs live under `/app/edgeprobes/`; a non-empty `EPT_DATA_DIR` overrides reads and a non-empty `EPT_AUDIT_DIR` overrides writes, otherwise those defaults apply. Create the audit directory when missing and never rewrite the evidence tree.
