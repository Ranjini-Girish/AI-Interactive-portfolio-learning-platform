Reliability teams need a frozen snapshot of circuit breaker posture for every upstream-backed service after rolling failure windows, tier thresholds, incident-driven deltas, staged failure-day suppressions, gold-only upstream degradation penalties, fleet-wide silver surge pressure, and explicit operator force directives are all applied on the same timeline anchored to `pool_state.current_day`. The outcome is a reproducible audit trail a downstream system can ingest without reprocessing raw logs.

The system delivers five UTF-8 JSON documents under `/app/audit/` named `service_verdicts.json`, `tier_thresholds.json`, `incident_journal.json`, `upstream_touchpoints.json`, and `summary.json`. Field names, ordering rules, incident kinds, and every numeric relationship tied to the pool day are defined in `/app/breakers/SPEC.md`, including the canonical JSON presentation each artifact must follow.

Technical formatting and paths: bundled evidence lives under `/app/breakers/`. A non-empty `CBSA_DATA_DIR` overrides the input root; a non-empty `CBSA_AUDIT_DIR` overrides the output directory. When a variable is unset or empty, that side falls back to `/app/breakers/` for reads and `/app/audit/` for writes. The output directory is created when missing. The default `/app/breakers/` tree is not modified by the audit run.
