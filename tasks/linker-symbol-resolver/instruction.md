A TypeScript on Node 22 linker symbol resolver is scaffolded at `/app/` with stub source files under `/app/src/` and headers in `/app/include/`. Build with `make -C /app` to produce `/app/build/linker`, then run the binary to generate `/app/output/link_report.json`.

The tool loads JSON-format "object files" listed in `/app/data/link_config.json`, resolves cross-file symbol references, merges like-named sections across objects (aligned per each contribution's declared alignment), and applies relocations. GLOBAL definitions override WEAK ones; multiple GLOBAL definitions of the same symbol produce an error; unresolved references produce an error. LOCAL symbols are visible only within their object file. `R_ABS_32` relocations compute `symbol_address + addend`; `R_PC_32` computes `symbol_address + addend - relocation_site_address`. See `/app/data/docs/ARCHITECTURE.md` for the output schema. Do not modify files under `/app/data/`.
