A Kerberos realm's keytab-modification history, ticket-request log, principal directory, rotation policy, and enctype policy are spread across the JSON and JSONL files under `/app/data/`: `/app/data/pool_state.json`, `/app/data/policies/rotation_policy.json`, `/app/data/policies/enctype_policy.json`, per-principal `/app/data/principals/<name>.json`, and append-only logs `/app/data/events/keytab_chunk_*.jsonl` and `/app/data/events/tgs_chunk_*.jsonl`. The complete contract — schema validation rules, kvno-lifecycle bookkeeping, active and cross-fade windows, rotation compliance, ticket-validity verdicts, anomaly classification with severity, and the exact schema of every required artifact — is documented in `/app/data/SPEC.md`; read that file before producing output. Produce exactly five JSON files under `/app/audit/`: `/app/audit/kvno_lifecycle.json`, `/app/audit/rotation_compliance.json`, `/app/audit/ticket_validity.json`, `/app/audit/anomalies.json`, and `/app/audit/summary.json`. The dataset under `/app/data/` is read-only and must not be modified.

All outputs must be UTF-8, indented with two spaces, with object keys emitted in sorted order at every level, and end in a single trailing newline; every list field must follow the sort order specified in `/app/data/SPEC.md`, so that two correct implementations of the same contract produce byte-identical files for the same input.
