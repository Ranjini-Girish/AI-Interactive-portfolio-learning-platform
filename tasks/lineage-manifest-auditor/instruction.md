Data governance needs a frozen read-only reconciliation of artifact manifests and their declared lineage edges against offline signing policy, digest hygiene, and the same-day incident playbook (quarantine lists plus depth caps). The audit must be runnable repeatedly without mutating the bundled tree so downstream consumers can diff JSON snapshots. The bundled tree already contains the manifest catalog, pool day, incident envelopes, and frozen policy tables referenced by the normative spec.

Produce five UTF-8 JSON files under `/app/audit/` named `manifest_graph.json`, `integrity_report.json`, `policy_screen.json`, `incident_journal.json`, and `summary.json`. Every field name, ordering rule, incident filter, and canonical JSON encoding rule lives in `/app/lineage_manifests/SPEC.md`. Inputs sit under `/app/lineage_manifests/`. When `LMA_DATA_DIR` is non-empty it overrides the read root; when `LMA_AUDIT_DIR` is non-empty it overrides the write root; otherwise use `/app/lineage_manifests/` and `/app/audit/`. Create the audit directory when missing. Do not add, delete, or rewrite anything under the read root while producing outputs.
