The read-only registry under `/app/registry/` contains `/app/registry/pool_state.json`, `/app/registry/policy/policy.json`, `/app/registry/incidents/incident_log.json`, every `/app/registry/packages/*.json` package manifest, and `/app/registry/SPEC.md`. Read the specification file in full before writing code; it defines tag parsing, naming skew, JSON name collision grouping, incident winner selection, per-field effective severity ordering, package rollup histogram rules, the five JSON output shapes, every sorting convention, and canonical UTF-8 JSON with two-space indentation, sorted object keys at every depth, and a single trailing newline.

Implement in Go under `/app/src/` as `package main`, built to `/app/bin/tagmatrix`. The binary reads `GTA_REGISTRY_DIR` and `GTA_AUDIT_DIR`, defaulting to `/app/registry/` and `/app/audit/`, and with no arguments writes exactly `tag_parse_matrix.json`, `json_name_collisions.json`, `package_rollups.json`, `incident_resolution.json`, and `summary.json`. Do not modify `/app/registry/`. Those five names must be the only files in the audit directory after a successful run.
