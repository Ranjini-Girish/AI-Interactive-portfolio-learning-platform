Normative contract for UTF-8 JSON inputs under the data root only. All emitted audit files are UTF-8 JSON with two-space indentation, ASCII-only text, sorted object keys at every object depth, and no trailing newline after the closing brace or bracket.

Files: `catalog.json` lists string `days` (ISO `YYYY-MM-DD`) and string `rings`, both sorted ascending in the file. `policy.json` has positive integer `incident_suppress_floor`, non-negative integer `overrun_slack` (allowed occupancy before overrun), array `tiers` of objects with string `label` and positive integer `high_water_max` strictly ascending by `high_water_max` where the final object always uses `label` `breach` with a `high_water_max` large enough to cover every snapshot `high_water`. `incidents.json` has object `days` mapping day strings to objects with integer `severity` (>=0). Per snapshot path `snapshots/<day>/<ring>.json` with positive integer `capacity`, non-negative integers `producer_total` and `consumer_total`, non-negative integers `instrument_wrap_prod` and `instrument_wrap_cons`, non-negative integer `high_water`, integer `consumer_tail_lo`, and array `splice_windows` of objects with string `id`, integers `tick_lo` and `tick_hi`, non-negative integers `claimed_span` and `reported_wrap_parity` (only `0` or `1` accepted). Missing files or malformed JSON abort the audit (non-zero tool exit); treat absent `splice_windows` as empty array.

Half-open splice ticks use `[tick_lo, tick_hi)` so `tick_delta = tick_hi - tick_lo`. Let `occupancy = producer_total - consumer_total`. Overrun when `occupancy > capacity + policy.overrun_slack`. Predicted producer wraps `pred_wrap_prod = producer_total / capacity` using integer division. Predicted consumer wraps `pred_wrap_cons = consumer_total / capacity`. Let `expected_xor = (pred_wrap_prod ^ pred_wrap_cons) & 1` and `parity_bit = (instrument_wrap_prod ^ instrument_wrap_cons) & 1`. When `0 <= occupancy <= capacity + overrun_slack`, status is `match` if the bits are equal otherwise `mismatch`. When outside that band, status is `anomaly` and bits are still emitted numerically.

Per splice window: if `claimed_span != tick_delta` then `status` is `span_mismatch`. Else if `(consumer_tail_lo % capacity) != (consumer_total % capacity)` then `tail_desync`. Else if `(reported_wrap_parity & 1) != (((consumer_total % capacity) + claimed_span) / capacity % 2)` in integer arithmetic then `parity_drift`. Else `ok`. Sort `splice_inventory` entries by `day`, `ring`, `id`.

Incident suppression: a day is suppressed when `incidents.days[day].severity >= incident_suppress_floor`. Suppressed days are omitted entirely from `splice_inventory`, `overrun_ledger`, `wrap_parity_report`, and `watermark_tier_matrix` rows. `incident_suppression.json` lists `suppressed_days` sorted, includes integer `policy_floor`, and object `severity` mapping each suppressed day to its integer severity.

`overrun_ledger` rows for non-suppressed days only, sorted by `day`, `ring`, each with keys `day`, `ring`, `occupancy`, `status` either `nominal` when `occupancy <= capacity`, `slack` when `capacity < occupancy <= capacity+overrun_slack`, or `overrun` when strictly above `capacity+overrun_slack`.

`wrap_parity_report` rows sorted by `day`, `ring` with keys `day`, `ring`, `parity_bit`, `expected_xor`, `status` either `match` when expected equals instrument xor bit and occupancy in range, `mismatch` when in range but unequal, or `anomaly` when outside nominal occupancy band.

Watermark tiering walks `tiers` in file order (already ascending `high_water_max`) and selects the first tier whose `high_water_max` satisfies `high_water <= high_water_max`; the breach tail must be wide enough that every snapshot still matches some tier. Rows sorted by `day`, `ring` with `day`, `ring`, `high_water`, `tier`.

Emit exactly five files under the audit root: `splice_inventory.json` with object key `entries` array; `overrun_ledger.json` with `rows`; `wrap_parity_report.json` with `rows`; `incident_suppression.json` with `policy_floor`, `severity`, `suppressed_days`; `watermark_tier_matrix.json` with `rows`. Every object uses sorted keys as above.

Canonical encoding for hashing nested values uses `json.dumps(value, sort_keys=True, separators=(",", ":"), ensure_ascii=True)` plus a single newline suffix for digest helpers in tests.
