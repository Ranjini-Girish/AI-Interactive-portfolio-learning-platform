# VFS shadow merge audit (normative)

All paths below are rooted at `/app/vfs_shadow/`. The auditor reads every JSON file under `mounts/`, `inodes/`, `incidents/`, and `snapshots/`, plus the top-level `pool.json` and `tiers.json`. Evaluation instants and epoch math use integers only; division is floor toward negative infinity only where stated as floor; here all operands are non-negative.

Pool day index is `max(0, floor((as_of_epoch - anchor_epoch) / day_length_sec))` using fields from `pool.json`. Shadow bytes for a mount apply only when `shadow_day_lo <= pool_day_index <= shadow_day_hi`. Shadow decay multiplies the shadow term only (never base bytes): let `lag = pool_day_index - last_touch_pool_day`. If `lag <= 0`, decay is `1`. Otherwise decay is `max(0, shadow_stale_span_days - lag) / shadow_stale_span_days` using `shadow_stale_span_days` from `pool.json`. The shadow adjustment is `floor(shadow_bytes * shadow_weight * decay)` where `shadow_weight` comes from the mount tier entry in `tiers.json`. If any active incident selects `freeze_merge` for a mount, that mount’s shadow adjustment is forced to `0` (base bytes unchanged). Active incident means `start_epoch <= as_of_epoch < end_epoch`, the incident `pool_id` equals the mount `pool_id`, and the incident scope matches the mount: `scope_kind` `pool_wide` matches every mount in that pool; `mount` matches when `scope_mount_id` equals the mount id; `subtree` matches when the mount id equals `scope_mount_id` or the mount is a descendant of `scope_mount_id` in the parent tree. Among active incidents for a mount, pick the one with greatest `precedence`; tie-break by lexicographically smallest `incident_id`.

Inode rows appear across all `inodes/*.json` files (merge arrays). Each row has `inode_id`, `home_mount_id`, `bytes`, and optional `borrow_target`. Follow `borrow_target` pointers until absent, detecting cycles by revisiting a node on the walk. If a cycle exists, the row is `broken`; its bytes are billed only to `home_mount_id` and are doubled for quota totals on that home mount. If no cycle, billing mount is the `home_mount_id` of the terminal inode reached. Sum `bytes` (after doubling when broken) into `inode_quota_to_mount[billing_mount]`.

Build the mount tree using `parent_mount_id`, with `null` or empty meaning none. Sort children by `(snapshot_seq, mount_id)` where `snapshot_seq` comes from `snapshots/*.json` merged as a map keyed by mount id defaulting `0`. Post-order compute `merged_total`: start `post_shadow = base_bytes + shadow_adjustment`. Then add `inode_quota_to_mount[mount_id]`. Then add the sum over ordered children of `min(child.merged_total, child.export_cap_bytes)`; `export_cap_bytes` is required on every mount file.

Emit five UTF-8 JSON documents under `/app/audit/`, minified with stable key order as produced by the reference emitter: `summary.json`, `merge_graph.json`, `cap_violations.json`, `inode_report.json`, `freeze_state.json`. `summary.json` fields: `pool_day_index` (int), `mount_count` (int), `inode_row_count` (int), `active_incident_count` (distinct incidents active for at least one mount), `freeze_merge_mount_count` (mounts whose winning incident forces `freeze_merge`), `cap_violation_count` (violations list length), `max_merged_total` (max merged_total across mounts). `merge_graph.json` has `nodes` array sorted by `mount_id`, each with `mount_id`, `parent_mount_id` (string or null), `post_shadow`, `rolled_children`, `inode_quota`, `merged_total`. `cap_violations.json` has `violations` sorted by `mount_id`, each with `mount_id`, `tier`, `hard_cap`, `merged_total`. Include a violation whenever `merged_total > hard_cap` for that tier. `inode_report.json` has `rows` sorted by `inode_id` with `inode_id`, `billing_mount_id`, `broken`, `bytes_accounted`. `freeze_state.json` has `mounts` sorted by `mount_id` with `mount_id`, `winning_incident_id` (string or null), `freeze_merge` (bool).
