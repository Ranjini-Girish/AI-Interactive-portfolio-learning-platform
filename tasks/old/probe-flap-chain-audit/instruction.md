Operations needs a frozen health-probe verdict for every node in the dependency forest after tier-scaled fail, flap, and soak thresholds, tier-specific rolling-window spans, rolling-window transition counts, post-fail soak windows, flap-day and fail-day suppressions, per-node tier promotion logs, and incident-driven isolations and forced-unhealthy directives are evaluated against the pool day in `/app/probeflaps/pool_state.json`. The audit must also apply the cross-reference pressure overlay defined under `/app/probeflaps/links/` after every node's primary status is fixed, so that nodes reachable from a degraded source by a directed cross-reference edge are reclassified when their own primary status would otherwise be `healthy`. The fail-day suppression rule must reach both the effective failure count and the determination of the most recent unsuppressed fail day. The result must be deterministic JSON that automation can consume without re-reading raw node files.

Produce six UTF-8 JSON documents under `/app/audit/` named `node_verdicts.json`, `tier_policy.json`, `incident_journal.json`, `dependency_touchpoints.json`, `cross_ref_overlay.json`, and `summary.json`. Every field name, sort order, status enum, numeric rule tied to the pool day, audit-day effective-tier resolution rule, cross-reference overlay rule, and canonical JSON layout each file must use is defined in `/app/probeflaps/SPEC.md`.

Bundled inputs live under `/app/probeflaps/`. When `PFCA_DATA_DIR` is set to a non-empty value it replaces the input root; when `PFCA_AUDIT_DIR` is set to a non-empty value it replaces the output directory. If either variable is unset or empty, reads default to `/app/probeflaps/` and writes to `/app/audit/`. Create the output directory when it is missing. The bundled `/app/probeflaps/` tree must not be modified by the audit run.
