Build a deterministic Go binary at `/app/bin/modarb` that turns a frozen multi-module Go workspace into a five-file decision log: per-dependency version resolutions under minimum-version selection with exclusion and retraction gates, per-row replace-directive verdicts under a workspace quorum, vulnerability triage that accounts for severity gating and reachability symbols, and a cycle-and-retract report that preserves cycle-blocked modules and tool-only entries without losing their accounting. The plan must be reproducible from the workspace tree on every run.

Place Go sources under `/app/src/` and write five UTF-8 JSON artifacts under `/app/decisions/` named `version_resolution.json` (seven actions), `replace_directive_audit.json` (five statuses), `vulnerability_exposure.json` (six statuses), `cycle_and_retract_report.json`, and `summary.json` (fourteen integer counters). Grammar for the workspace manifest, modules, registry, advisories, governance policy, incident events, version comparator, five-phase pipeline, sort orders, and canonical JSON formatting lives in `/app/modgraph/SPEC.md`.

Inputs are under `/app/modgraph/`. When `GMBA_MODGRAPH_DIR` is set to a non-empty value it overrides the input root; when `GMBA_DECISIONS_DIR` is set to a non-empty value it overrides the output directory. Unset or empty variables fall back to `/app/modgraph/` for reads and `/app/decisions/` for writes. Create the output directory if needed. Do not modify files under the input root during the run.
