The read-only registry under `/app/registry/` holds current and baseline contracts in `services/` and `baselines/`, consumer data in `consumers/`, policy, incidents, `pool_state.json`, and `/app/registry/SPEC.md`. Read that file completely before coding; it is under ten kilobytes so one read should capture every rule. Its opening section states literals you must copy exactly, including the classification reason for force-break overrides and the rule that cyclic consumer impact reuses the same blocked-cycle membership computed for migration, not ad hoc walk revisits.

The specification defines diffs, tier-graded breaking, incident deduplication, propagation through direct reads and field maps, migration phases with forced and deferred cases, risk scoring, the five JSON output shapes, sort orders, and canonical encoding with two-space indentation, sorted keys at every object level, and a single trailing newline.

Implement in Go under `/app/src/` as `package main`, built to `/app/bin/auditor`. The binary reads `ODA_REGISTRY_DIR` and `ODA_AUDIT_DIR`, defaulting to `/app/registry/` and `/app/audit/`, and with no arguments writes exactly `change_classification.json`, `consumer_impact.json`, `migration_plan.json`, `risk_assessment.json`, and `summary.json`. Do not modify `/app/registry/`. Those five names must be the only files in the audit directory after a successful run.
