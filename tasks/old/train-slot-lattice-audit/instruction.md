Shared training clusters ration finite accelerator slots across research tenants. Each run submits a request with a priority rung, a slot count, and an optional dependency on another run id. Laboratory policy can freeze tenants, override rung ordering, and trim effective capacity when calendar incidents land before the evaluation day in `pool_state.json`. The platform team needs a reproducible read-only audit of how those rules ration slots once dependency cycles, tenant freezes, and capacity trims are all evaluated together. The audit must not rewrite any bundled evidence.

Ship five UTF-8 JSON documents under `/app/audit/` named `allocation_plan.json`, `dependency_graph.json`, `incident_trace.json`, `tenant_utilization.json`, and `summary.json`. Every structural rule, sort order, incident filter, allocation tie-break, and canonical JSON presentation is defined in `/app/tslattice/SPEC.md`, including how optional environment variables select alternate roots. When `TSL_DATA_DIR` is set to a non-empty string it replaces `/app/tslattice/` for reads; when `TSL_AUDIT_DIR` is set to a non-empty string it replaces `/app/audit/` for writes; otherwise use those default paths. The ancillary JSON files under the same tree are distractors only; they must not influence the audit. Create the audit directory when it is missing. Every file must be valid UTF-8 JSON with the byte-level layout described in the specification so downstream validators can compare artifacts without special casing.
