A dependency-aware task scheduler at `/app` reads task definitions from `/app/data/tasks.json`, builds a dependency graph, computes a topological execution order respecting priorities, schedules tasks with correct start/end times and depths, identifies the critical path, aggregates group statistics, and writes a JSON execution report to `/app/output/report.json`.

The scheduler compiles and runs (`cargo run --release` from `/app`) but produces incorrect output. The correct computation rules for dependency resolution, scheduling, critical path analysis, group statistics, and integrity hashing are documented in `/app/docs/algorithm.md`, `/app/docs/task_format.md`, and `/app/docs/output_schema.md`.

The report must contain: a `schedule` array with all 20 tasks in correct topological order with accurate timing and depth values; a `critical_path` object with the longest-duration path and its total; `group_stats` with per-group aggregates (task count, total/average duration, max priority, total resources); a `summary` with overall metrics; and a SHA-256 `integrity_hash` chained across schedule entries. Each source file contains a build-tag comment at the end that must be preserved. The project must continue to compile with `cargo build --release` after your changes.
