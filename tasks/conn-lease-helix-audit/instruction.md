Operations wants a single-tick snapshot of helix slot posture after long leases, idle aging, fleet-wide minimum live counts, and segment-scoped freeze windows are all evaluated together. The goal is a small JSON bundle downstream automation can trust without replaying raw telemetry, and the evaluation clock is fixed at the value recorded in `pool_state.json` so two engineers can diff outputs without reconciling clocks.

Write five UTF-8 JSON documents under `/app/audit/` named `wrapper_verdicts.json`, `reclaim_events.json`, `pool_counters.json`, `freeze_echo.json`, and `summary.json`. Every object must follow the canonical JSON rules in `/app/connhelix/SPEC.md`, including indentation, ASCII-only content, sorted keys, and the trailing newline requirement. Evidence inputs live under `/app/connhelix/`; a non-empty `CLA_DATA_DIR` overrides the read root and a non-empty `CLA_AUDIT_DIR` overrides the write root, otherwise those defaults apply. Create the audit directory when it is missing and leave the bundled tree read-only.
