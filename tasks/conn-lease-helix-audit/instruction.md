Operations wants a single-tick snapshot of helix slot posture after long leases, anchor renewals gated by channel binding and a distinct-channel renewal quorum, cascade fallout from leaked parent slots that demotes deep descendants to an orphan verdict, idle aging, fleet-wide minimum live counts, per-segment floor counts, and segment-scoped or wrapper-scoped freeze windows are all evaluated together. The goal is a small JSON bundle downstream automation can trust without replaying raw telemetry, and the evaluation clock is fixed at the value recorded in `pool_state.json` so two engineers can diff outputs without reconciling clocks.

Write five UTF-8 JSON documents under `/app/audit/` named `wrapper_verdicts.json`, `reclaim_events.json`, `pool_counters.json`, `freeze_echo.json`, and `summary.json`. Every object must follow the canonical JSON rules in `/app/connhelix/SPEC.md`, including indentation, ASCII-only content, sorted keys, and the trailing newline requirement. Evidence inputs live under `/app/connhelix/`; the renewal records under `anchors/` are part of the normative input and feed the effective-lease computation, with each wrapper carrying a `bound_channels` array that names the anchor channels permitted to renew it and `policy.renewal_quorum` controlling how many distinct channels must contribute an applicable renewal before the rescue takes effect. Cascade propagation honours `policy.cascade_depth_cap`, demoting deeper descendants to a separate verdict. `summary.json` carries both the `ignored_renewals` list and an `under_quorum_renewals` list, and `pool_counters.json` carries a `cascade_orphans` count. A non-empty `CLA_DATA_DIR` overrides the read root and a non-empty `CLA_AUDIT_DIR` overrides the write root. Create the audit directory when it is missing and leave the bundled tree read-only.
