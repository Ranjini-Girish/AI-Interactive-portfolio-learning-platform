A multi-tenant git-hook policy audit dataset is spread across the JSON files under `/app/repos/`: `pool_state.json`, `baseline.json`, `incident_log.json`, and per-repo files at `repos/<repo>/profile.json`, `repos/<repo>/hooks_installed.json`, and `repos/<repo>/recent_commits.json`. The complete contract — input validation rules, per-repo policy override semantics, tier-dependent commit-message review, time-bounded waivers, compromise-quarantine cascade, and the exact schema of every required artifact — is documented in `/app/repos/SPEC.md`; read that file before producing output. Produce exactly five JSON files under `/app/audit/`: `repo_compliance.json`, `hook_install_plan.json`, `commit_review.json`, `branch_protection.json`, and `summary.json`. The dataset under `/app/repos/` is read-only and must not be modified.

All outputs must be UTF-8, indented with two spaces, with object keys emitted in sorted order at every level, and end in a single trailing newline; every list field must follow the sort order specified in `/app/repos/SPEC.md`, so that two correct implementations of the same contract produce byte-identical files for the same input.
