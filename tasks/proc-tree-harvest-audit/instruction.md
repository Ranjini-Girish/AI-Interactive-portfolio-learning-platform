Build a deterministic Java 21 tool that replays a synthetic POSIX process tree. Read `/app/data/processes.json` (the initial process set, each with an integer `pid`, an integer `ppid` of `0` for roots, an integer `uid`, a string `state` that is always `"RUNNING"` initially, an integer `start_tick`, and a string `cmdline`), `/app/data/events.json` (a strictly increasing dense `seq` log of `fork`, `exit`, `wait`, `kill`, and `exec` operations), and `/app/data/policy.json` (with `init_pid` (positive integer), `orphan_handling` in `{"reparent_to_init","leave_orphaned"}`, `implicit_init_harvest` (bool), `wait_on_living_child` in `{"diagnostic","noop"}`, and `track_lineage` (bool)).

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/proctreehv` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

Write five JSON files into the second-argv output directory (the verifier's primary run uses `/app/output/`): `process_state.json` (the final process set sorted by `pid`, each entry carrying its `cmdline`, optional `exit_code`, optional `exit_signal`, optional `exit_tick`, `pid`, `ppid`, `start_tick`, terminal `state` of `"RUNNING"`, `"ZOMBIE"`, or `"EXITED"`, and `uid`), `harvest_log.json` (chronological list of every zombie harvest, each entry with the harvester's `parent_pid`, the harvested `pid`, the trigger of `"wait"` or `"init_harvest"`, and `seq`/`tick`), `process_diagnostics.json` (per-event diagnostics drawn only from the closed code set documented under `/app/docs/diagnostics.md`, sparse and sorted by `seq`, then within an event by `(severity_rank, code, pid)` where `null` pid sorts before all integers), `lineage_graph.json` (the parent-to-child DAG built only when `policy.track_lineage` is true, with every pid ever observed as a node and edges typed `"fork"` or `"reparent_init"`, deduplicated and sorted), and `summary.json` (the documented counters and the sorted ascending list of distinct `uid` integers belonging to processes still `RUNNING` at trace end). Every output must be canonical JSON: UTF-8, ASCII-only, two-space indent, lexicographically sorted object keys at every depth, and a single trailing newline. The verifier reruns a reference implementation against the same `/app/data/` and compares the five structures byte-for-byte, so do not commit a prebuilt binary, do not hardcode answers, and the Java 21 build must actually run.
