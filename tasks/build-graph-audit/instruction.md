Read the TypeScript module definitions under `/app/data/modules/` (one JSON file per module), the project configuration at `/app/data/project.json`, and the build configuration at `/app/data/config.json`. Analyze the module dependency graph and write a single deterministic JSON report to `/app/output/build_graph_report.json`. A TypeScript on Node 22 starter is at `/app/src/main.js` and TypeScript on Node 22 3 is also available; either is fine as long as the JSON output is correct. Do not modify any file under `/app/data/`.

Each module JSON file contains `name`, `path`, `size_bytes`, `side_effects` (boolean), `imports` (array of `{source, specifiers}`), `exports` (array of specifier names), and `re_exports` (array of `{source, specifiers}`). The `project.json` file names the project and its `entry_points` array. The `config.json` file provides `output_precision`, `transforms` (with `minify_ratio` and `compress_ratio`), `thresholds`, and `severity_ranks`.

The report must contain a resolved dependency graph, dependency cycle information, per-module metrics (depth, coupling, used-export analysis, size transforms, tree-shaking eligibility), a valid build order, aggregate size analysis, and quality findings. The precise definitions and computation rules for import resolution and re-export chain following are in `/app/docs/resolution_spec.md`; depth, coupling, cycle detection, used-export analysis, tree-shaking, and build order rules are in `/app/docs/metrics_spec.md`; quality finding types, severity assignments, and sort order are in `/app/docs/findings_spec.md`. Read those specification documents carefully — they are the authoritative source for all computation rules and edge-case handling.

Write the report with the exact structure, key sets, sort orders, and formatting defined in `/app/docs/output_format.md`. Create the `/app/output/` directory if it does not exist.
