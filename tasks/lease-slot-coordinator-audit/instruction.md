Platform operations need a frozen lease-coordination snapshot for every host-slot pairing after tier-scaled grace and renewal caps, rolling renewal totals, contested-slot witness checks that only credit co-tenant attestations on the same slot, and incident-driven freezes, compromises, and forced expirations are evaluated against the pool day in `/app/leases/pool_state.json`. Witness attestation windows can use a policy witness-day lower bound that is separate from the renewal rolling window lower day. Each lease verdict row records how many deduplicated witness pairs were credited whenever witness sufficiency is evaluated for that row. The result must be deterministic JSON that downstream automation can consume without re-reading raw host files.

Produce five UTF-8 JSON documents under `/app/audit/` named `lease_verdicts.json`, `tier_policy.json`, `incident_journal.json`, `slot_contention.json`, and `summary.json`. Every field name, sort order, status enum, and numeric rule tied to the pool day is defined in `/app/leases/SPEC.md`, including the canonical JSON layout each file must use.

Bundled inputs live under `/app/leases/`. When `LSCA_DATA_DIR` is set to a non-empty value it replaces the input root; when `LSCA_AUDIT_DIR` is set to a non-empty value it replaces the output directory. If either variable is unset or empty, reads default to `/app/leases/` and writes to `/app/audit/`. Create the output directory when it is missing. The bundled `/app/leases/` tree must not be modified by the audit run.
