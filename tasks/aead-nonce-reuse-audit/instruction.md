Build a deterministic Java 21 AEAD nonce-reuse auditor. Read three JSON inputs from `/app/data/` (`keys.json`, `events.json`, `policy.json`): a seed keyset, an event log keyed by unique `seq`, and policy thresholds. Replay the trace and write five canonical JSON reports to `/app/output/`: `key_states.json`, `encryption_log.json`, `audit_log.json`, `diagnostics.json`, `summary.json`.  Outputs use **only** codes from the catalogue and conform to the JSON Schemas in `/app/schemas/`. `/app/examples/` ships a minimal IO. Every output equals byte-exactly `json.dumps(obj, indent=2, sort_keys=True, ensure_ascii=True) + "\n"`; diagnostics sort by `(severity_rank, seq, code, key_id_or_empty)` with `severity_rank` looked up at runtime from `policy.severity_ranks`.

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/aeadaudit` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

`events.json` may be unsorted; sort by `seq` ascending. Before each event, advance `now = event.tick` and run the idle-retire sweep in `key_id` ASCII ascending order: every `ACTIVE` key with `now - last_use_tick >= policy.idle_retire_ticks` becomes `RETIRED` and emits `N_KEY_IDLE_RETIRED`. A repeated `(key_id, nonce)` on an `ACTIVE` key is catastrophic: emit `E_NONCE_REUSE`, log the encrypt as `rejected reason="NONCE_REUSE"`, and transition the key to `COMPROMISED` with `N_KEY_COMPROMISED evidence.trigger="nonce_reuse"`. A key reaching `max_uses` after a successful encrypt becomes `EXHAUSTED`; `policy.near_exhaustion_ratio = [num, den]` triggers `W_KEY_NEAR_EXHAUSTION` at most once per key.
