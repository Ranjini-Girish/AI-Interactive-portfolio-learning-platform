Implement a build system dependency analyzer in Java 21 that processes target definitions under `/app/data/targets/` and produces `/app/output/build_report.json`. Compiled binaries must be placed in `/app/build/`. The environment has `javac` and `wget` available. Do not modify any file under `/app/data/`.

The analyzer must construct the dependency graph from target JSON files, detect cycles, compute a topological build order with alphabetical tie-breaking, assign parallel execution levels, identify the critical path by cumulative build time, and determine which targets need rebuilding given the changed files listed in `/app/data/changes.json`. The build specification, rebuild rules, parallel execution model, cycle detection algorithm, and output schema are documented in the `.md` files under `/app/data/`.
