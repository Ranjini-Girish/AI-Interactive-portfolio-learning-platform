The read-only reliability dataset is under `/app/signals/` and includes service manifests, per-service time windows, incident history, on-call rotations, and policy files. The complete processing contract is in `/app/signals/SPEC.md`; it defines event acceptance, severity derivation, dependency-impact propagation, node-state composition, paging priority, and the exact schema and sort order of every output field. Read that specification first and follow it exactly.

Implement the solution in Go. Place source code under `/app/src/` with at least one file declaring `package main`, compile one executable to `/app/bin/auditor`, and run that binary with no arguments. The binary must read input and output locations from `GIA_SIGNALS_DIR` and `GIA_REPORT_DIR`, defaulting to `/app/signals/` and `/app/report/` when unset. Execution must produce exactly five files in the report directory: `anomaly_report.json`, `blast_radius.json`, `paging_plan.json`, `node_health.json`, and `summary.json`. Do not modify files under `/app/signals/`.

All produced JSON must be UTF-8, pretty-printed with two-space indentation, with object keys sorted at every level, and with exactly one trailing newline.
