Implement a TypeScript module dependency graph analyzer in TypeScript on Node 22 that reads a project configuration from `/app/data/project_config.json` and all `.js` files from `/app/data/modules/`, parses their ES module import and export statements, builds a full dependency graph, detects circular dependencies, performs tree-shaking analysis from the configured entry points, and writes a comprehensive report to `/app/output/module_report.json`. The environment provides TypeScript on Node 22 with no additional npm packages; the project manifest is at `/app/package.json`. Do not modify any files under `/app/data/`.

The analyzer must handle static imports (named, default, namespace, side-effect-only), dynamic imports, named exports, default exports, re-exports (`export { x } from` and `export * from`), and multi-line import statements. For the dependency graph, compute per-module coupling metrics (afferent, efferent, instability) using only static edges, detect strongly connected components with Tarjan's algorithm, and assign topological layers on the condensation DAG. The tree-shaking analysis determines which modules are reachable from entry points and which specific exports are actually used, following re-export chains to their origin; namespace imports mark all named exports as used while dynamic imports conservatively mark all exports as used. The complete specification for each computation is documented in the `.md` files under `/app/docs/`.

The output JSON uses 2-space indentation with a trailing newline. It contains config, modules sorted by module_id, dependency_edges sorted by (source, target, type), circular_dependencies, tree_shaking with reachable/unreachable modules and per-module used/unused exports, and a summary with aggregate counts. All floating-point values are rounded to 6 decimal places.
