Platform ML needs a frozen readout of feature-store slice freshness that reconciles declared training tiers against row-count capacity floors, day-based staleness budgets, lineage lag against parent slices, and cluster-wide compromise quarantine without rewriting the evidence tree. Each slice record carries a cluster name, declared tier, refresh day, optional lineage parent, and row count; incidents may force a tier label or quarantine an entire cluster from a compromise effective day.

Emit five UTF-8 JSON files under `/app/audit/` named `slice_profiles.json`, `cluster_rollups.json`, `lineage_events.json`, `capacity_summary.json`, and `summary.json`. Canonical JSON rules, tier vocabulary, freshness precedence, lineage grace, and incident semantics are all defined in `/app/featstore/SPEC.md`. Inputs live under `/app/featstore/`; a non-empty `FSF_DATA_DIR` overrides reads and a non-empty `FSF_AUDIT_DIR` overrides writes, otherwise those defaults apply. Create the audit directory when missing and never rewrite the evidence tree.
