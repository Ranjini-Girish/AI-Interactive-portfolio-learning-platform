Analytics operations needs a deterministic same-day export batch plan for every upstream source after batch dependency rules, tier-and-source sync windows, per-warehouse export caps with incident-driven bumps, shared warehouse credit pools with grant events, per-tier credit ceilings, batch embargoes, source compromise holds, and warehouse freeze directives are evaluated against `pool_state.current_day`. The outcome must be replayable without modifying the frozen input tree.

Write five UTF-8 JSON artifacts under `/app/audit/` named `source_plan.json`, `warehouse_ledger.json`, `batch_matrix.json`, `incident_journal.json`, and `summary.json`. Grammar for sources, batches, warehouses, incidents, blocked-candidate reasons, source status values, and canonical JSON formatting lives in `/app/export_batches/SPEC.md`.

Inputs live under `/app/export_batches/`. When `EBWA_DATA_DIR` is set to a non-empty value it overrides the input root; when `EBWA_AUDIT_DIR` is set to a non-empty value it overrides the output directory. Unset or empty variables fall back to `/app/export_batches/` for reads and `/app/audit/` for writes. Create the output directory if needed. Do not modify files under the input root during the audit run.
