# Leaky Bucket Shaper Replay

Build a deterministic C++17 byte-based leaky-bucket traffic shaper. Read `/app/data/buckets.json` (each bucket has a unique `bucket_id` `^[A-Za-z0-9._-]{1,32}$`, `capacity_bytes` ∈ `[1,10^9]`, `leak_bytes_per_tick` ∈ `[0,10^9]`), `/app/data/events.json` (dense `seq` from `0`; `type` ∈ `{"submit","tick","reconfigure"}` with exactly the documented fields—any non-dense `seq`, unknown `type`, missing/extra field, bad `bucket_id`, or unknown bucket on `reconfigure` is malformed), `/app/data/policy.json` (`count_dropped_bytes`, `track_admits`). `now_ticks` starts `0`, `+1` per `tick`. `submit`: if `current_bytes+size_bytes ≤ capacity_bytes`, add bytes (`N_ADMITTED`, log dispatch); unknown bucket → `E_UNKNOWN_BUCKET`; else drop fully (`W_DROPPED_OVERFLOW`, count overflow, add bytes to `dropped_bytes_total` only when `count_dropped_bytes`). `tick`: increment `now_ticks`, then drain every bucket by `min(current_bytes, leak_bytes_per_tick)`. `reconfigure`: equal cap+leak → `W_RECONFIG_NOOP`; else apply, and if new capacity < current level, clip and emit `W_CAPACITY_REDUCED` with `oldlevel->newcap`. Diagnostics use only the closed set in `/app/docs/diagnostics.md`; schemas plus `/app/docs/` are the closed contract.

Place `.cpp`/`.hpp` only under `/app/include/` and `/app/src/` (no `.h`/`.cc`/`.cxx` anywhere under `/app/`); ≥200 source lines. Build with `g++ -std=c++17` (structured GCC stamp in `.comment`, linking `libstdc++.so.6`) into native ELF `/app/build/shaper` whose mtime ≥ every `/app/` source mtime. Invoke with two positional args: input dir, output dir. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/include/`, `/app/docs/`, `/app/schemas/`, `/app/examples/` are immutable; writes outside `/app/output/` (incl. `/tmp`) are forbidden. `/app/data/` holds exactly the three named files.

Write four ASCII canonical-JSON files into `/app/output/`: `bucket_state.json` (sort by `bucket_id`), `admits.json` (sort `(seq, bucket_id)`; `{"admits": []}` when `track_admits=false`), `shaper_diagnostics.json` (sort `(seq, -severity_rank, code, bucket_id, detail)`), `summary.json` (`max_seq` `null` if no events; `admits_total` always counts even with `track_admits=false`). Canonical JSON: UTF-8/ASCII, 2-space indent, lex-sorted keys, no extras, single trailing `\n`. The verifier hash-pins inputs, wipes `/app/output/`, reruns the binary, recomputes a reference, and compares byte-for-byte.
