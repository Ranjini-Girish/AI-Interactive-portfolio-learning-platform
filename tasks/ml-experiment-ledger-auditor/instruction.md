The experiment ledger is read-only JSON under `/app/ledger/`: pool and governance roots, the incident stream, datasets, runs, checkpoints, and registry entries. Read `/app/ledger/SPEC.md` for the derivation contract, and read `/app/ledger/reason_contract.txt` whenever you need the exact lowercase one-line reason templates for run rows and unstable-checkpoint rows, because those strings are hashed byte-for-byte in the verifier. The short contract file restates only those templates so a partial read of the longer specification cannot silently drop them.

Implement the auditor in Go under `/app/src/` with `package main`, compiling to `/app/bin/ml-ledger-auditor`. The binary reads `MEL_DATA_DIR` and `MEL_AUDIT_DIR`, defaulting to `/app/ledger` and `/app/audit` when unset. With no arguments it writes exactly five UTF-8 JSON files in the audit directory: `run_status.json`, `lineage_graph.json`, `checkpoint_disposition.json`, `registry_promotion.json`, and `summary.json`. Each uses two-space indentation, sorted object keys at every level, list ordering exactly as in `SPEC.md`, and one trailing newline so correct solutions match byte-for-byte. Do not modify `/app/ledger/`.
