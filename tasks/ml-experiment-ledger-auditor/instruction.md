The experiment ledger is read-only JSON under `/app/ledger/`: pool state, governance config, incidents, datasets, runs, checkpoints, and registry entries. Read `/app/ledger/SPEC.md` for the derivation contract. Read `/app/ledger/reason_contract.txt` for the lowercase one-line reason templates that apply to run rows and unstable-checkpoint rows when the specification names those phrases; that short file exists so the literals are not lost when someone reads only part of SPEC.

Implement the auditor in Go under `/app/src/` with `package main`, compiling to `/app/bin/ml-ledger-auditor`. The binary reads `MEL_DATA_DIR` and `MEL_AUDIT_DIR`, defaulting to `/app/ledger` and `/app/audit` when unset. With no arguments it writes five UTF-8 JSON files in the audit directory: `run_status.json` for per-run classification; `lineage_graph.json` for dataset compromise propagation, depth, and downstream runs; `checkpoint_disposition.json` for keep-or-quarantine per checkpoint; `registry_promotion.json` for registry promotion against governance; `summary.json` for totals and sorted ids as in `SPEC.md`. Each uses two-space indentation, sorted object keys everywhere, list order per `SPEC.md`, and one trailing newline so matching implementations emit identical bytes. Do not modify `/app/ledger/`.
