A small product team's crash corpus is spread across the read-only JSON files under `/app/dumps/`: `pool_state.json`, `triage_config.json`, `module_map.json`, `incident_log.json`, per-release `releases/<version>.json`, and per-crash `crashes/<id>.json`. The complete contract — input validation rules, canonical signature derivation, release-window attribution, severity escalation, owner-assignment precedence, and the exact schema of every required artifact — is documented in `/app/dumps/SPEC.md`; read that file before producing output. Produce exactly five JSON files under `/app/triage/`: `cluster_index.json`, `attribution_report.json`, `severity_ranking.json`, `owner_assignment.json`, and `summary.json`. The dataset under `/app/dumps/` is read-only and must not be modified.

All outputs must be UTF-8, indented with two spaces, with object keys emitted in sorted order at every level, and end in a single trailing newline; every list field must follow the sort order specified in `/app/dumps/SPEC.md`, so that two correct implementations of the same contract produce byte-identical files for the same input.
