Bench engineers split a long discrete waveform into ordered part files, then replayed day-bounded incidents that zero individual samples or raise the saturation cap before a fixed-width sliding maximum is taken. Calibration tooling needs the capped waveform, per-window diagnostics tying raw peaks to the cap, and a compact summary that records how many output samples were produced after the concatenated vector was mutated.

Read-only inputs live under `/app/wmc_lab/`; `/app/wmc_lab/SPEC.md` defines part concatenation, incident eligibility, window width, cap handling, and the four UTF-8 JSON artifacts under `/app/wmc_audit/`: `dilated_series.json`, `window_trace.json`, `incident_trail.json`, and `summary.json`. Implement the audit in Go 1.23 with `go.mod` and sources under `/app/wmc_tool/`, build a single `wmcap` binary in that directory, and write only into `/app/wmc_audit/`. Non-empty `WMC_DATA_DIR` overrides the input root and non-empty `WMC_AUDIT_DIR` overrides the output directory; otherwise reads use `/app/wmc_lab/` and writes use `/app/wmc_audit/`. Create the output directory when missing. Match the spec’s canonical JSON presentation: ASCII-only text, two-space indentation, recursively sorted object keys at every object level, no stray trailing spaces on lines, and exactly one trailing newline at EOF.
