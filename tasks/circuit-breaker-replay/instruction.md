Two semantic points worth restating because agents miss them. First, an OPEN endpoint never trips back to HALF_OPEN on a `request` event, only on a later `tick` event whose new global tick is at least `tick_entered_open + recovery_ticks`; until then every request to an OPEN endpoint is short-circuited and the request_log records `state_at_admission == "OPEN"` and `admission == "short_circuited"` without touching the sliding window. Second, in HALF_OPEN every request is admitted as a probe, the very first probe failure or timeout drops the endpoint back to OPEN with the probes counters reset, and the endpoint only returns to CLOSED once `half_open_max_probes` consecutive probe successes have been observed; until then it stays HALF_OPEN.
