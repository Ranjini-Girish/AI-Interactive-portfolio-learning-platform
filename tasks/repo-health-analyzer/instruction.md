Implement a TypeScript repository health analyzer at `/app/src/main.js` (run via `TypeScript on Node 22`). Read commits from `/app/data/commits/` (JSON: hash, author_name, author_email, parent_hashes, timestamp, message, file_changes), branches from `/app/data/branches.json`, tags from `/app/data/tags.json`, mailmap from `/app/data/mailmap.json`, config from `/app/data/config.json`. Output: `/app/output/repo_health_report.json`.

Normalize emails: lowercase, then apply mailmap aliases transitively until stable (case-insensitive match). Display name per canonical email: from their latest commit. Depth: longest DAG path from any parentless root; octopus merges (≥3 parents) use second-highest parent depth + 1. Merge commits (≥2 parents) count 0.5 toward commit_count, 1.0 toward merge_count. File churn: process commits in reverse chronological order (ties: hash descending). Per-file base churn = `Math.floor(raw * (1 + depth * 0.05))`, raw = additions + deletions (text) or binary_churn_weight (binary, both null); binary_churn_weight from config. Each file tracks running_churn; contribution = base + `Math.floor(running_churn * 0.1)`, then add contribution to running_churn. After processing each commit's files, apply global decay: subtract `Math.floor(running_churn * 0.03)` from every tracked file not modified in that commit (minimum zero). Total churn = final running_churn. Author total_additions/total_deletions: raw sums, no scaling. Hotspot: `round6(Math.floor(total_churn * recency) * distinct_authors)`, recency = `Math.exp(-days_exact / half_life_days)`. days_since_last_change: floor of fractional days. Population stddev (÷N) for author stats. Review coverage: non-merge commits with line-start "Reviewed-by:", ÷ total non-merge; merges excluded from both.

Report: metadata (analysis_date, default_branch, total_commits, total_authors, total_files), authors sorted commit_count desc/first_commit_date asc/name asc (name, canonical_email, commit_count, merge_count, first_commit_date, last_commit_date, files_touched, total_additions, total_deletions), author_statistics (mean_commits, stddev_commits, median_commits), files with total_churn ≥ config min_churn, sorted hotspot_score desc/total_churn desc/path asc (path, total_churn, distinct_authors, last_change_date, days_since_last_change, recency_weight, hotspot_score, is_binary), branches (name → head_commit + depth), review_coverage (reviewed_count, total_non_merge, coverage_ratio), commit_depths (hash → depth). Sorted keys all levels, 2-space indent, trailing newline.
