A data-quality bundle of tab-separated tables sits read-only at `/app/tab_bundle/` together with `catalog.json`, `policy.json`, and the binding `SPEC.md`. The catalog lists which TSV files participate, the policy supplies extra missing literals, columns to skip from rollups, and the key column lists for two independent duplicate-detection systems, and the spec defines the schema-union behaviour, the missingness predicate, the global-index walk, the truncated six-digit percentage strings, and the exact JSON layout for the report. Read all four artefacts and assume the per-table headers may differ in which kept columns they include; a column that is absent from a table contributes a missing entry for every one of that table's data rows.

Build a Rust Cargo workspace rooted at `/app` using the toolchain already in the image. Compile with `cargo build --release` so the tool is produced at `/app/target/release/tabgap`. Invoking that binary with no arguments must perform the full audit and write exactly one regular file `/app/audit/gap_report.json`, leaving everything under `/app/tab_bundle/` byte-identical. The JSON must be UTF-8 with two-space indentation, sorted object keys at every level, ASCII-only string content, and a single trailing newline; two correct runs must be byte-identical. Exit status zero is required on the bundled inputs.
