Build a deterministic C++17 tool that replays mutex acquire and release events and records wait-for edges. Read `/app/data/mutexes.json` (initial mutexes, each with string `name` and nullable string `owner`), `/app/data/events.json` (a dense `seq` log of `tick`, `acquire`, `release`, and `try_acquire` operations with non-negative `tick` values), and `/app/data/policy.json` (`fifo_waiters` bool, `detect_cycles` bool, and `note_on_tick` bool). Place headers under `/app/include/`, sources under `/app/src/`, build with `g++ -std=c++17` via the project Makefile so the installed binary is `/usr/local/bin/mtxgraph`, and invoke it with exactly two positional arguments: the input directory then the output directory. Exit non-zero on wrong arity, missing inputs, or malformed JSON. Never modify `/app/data/`.

Semantic rules, diagnostic codes, per-field sorting, and canonical JSON requirements are documented under `/app/docs/`. Input JSON Schemas are `mutexes_input.schema.json`, `events_input.schema.json`, and `policy_input.schema.json`; output field shapes are in `/app/docs/output_format.md` with `summary.schema.json` for counters. `/app/examples/` shows minimal shapes. Write five files into the output directory: `mutex_state.json` (final mutexes sorted by `name`), `wait_edges.json` (active waiter-to-owner edges sorted by `(waiter, mutex)`), `action_log.json` (successful acquire, try_acquire, release, and wake rows chronological by `seq`), `diagnostics.json` (sparse per-event diagnostics from the closed code set in `/app/docs/diagnostics.md`, events sorted by `seq`, diagnostics within an event sorted by `(severity_rank, code, mutex)` with `null` mutex before any string), and `summary.json` (documented counters). Every output file must be canonical JSON: UTF-8, ASCII-only, two-space indent, lexicographically sorted keys at every depth, and a single trailing newline. The verifier rebuilds your binary, runs it on `/app/data`, and compares outputs byte-for-byte against a live reference, so do not ship prebuilt binaries or hand-written reports.
