Build a deterministic C++17 tool that replays mutex acquire and release events and records wait-for edges. Read `/app/data/mutexes.json` (initial mutexes, each with string `name` and nullable string `owner`), `/app/data/events.json` (a dense `seq` log of `tick`, `acquire`, `release`, and `try_acquire` operations with non-negative `tick` values), and `/app/data/policy.json` (`fifo_waiters` bool, `detect_cycles` bool, and `note_on_tick` bool). Place headers under `/app/include/`, sources under `/app/src/`, build with `g++ -std=c++17` via the project Makefile so the installed binary is `/usr/local/bin/mtxgraph`, and invoke it with exactly two positional arguments: the input directory then the output directory. Exit non-zero when the argument count is not exactly two, when any of the three named input files is missing, when input JSON is syntactically invalid, when an input fails its JSON Schema (a required field missing or of the wrong type, `tick` less than zero, `seq` values not dense from `0` to `N-1`, or `op` outside the closed set `{tick, acquire, release, try_acquire}`), or when an `acquire`, `release`, or `try_acquire` event carries a null `mutex` or `task`. Schema-validation failures abort the run before any output file is finalized, so a rejected input must not leave a complete five-file output set behind. Never modify `/app/data/`.

Semantic rules, diagnostic codes, per-field sorting, and canonical JSON requirements are documented under `/app/docs/`. Input JSON Schemas are `mutexes_input.schema.json`, `events_input.schema.json`, and `policy_input.schema.json`; output field shapes are in `/app/docs/output_format.md` with `summary.schema.json` for counters. `/app/examples/` shows minimal shapes. Write five files into the output directory: `mutex_state.json` (final mutexes sorted by `name`), `wait_edges.json` (active waiter-to-owner edges sorted by `(waiter, mutex)`), `action_log.json` (successful acquire, try_acquire, release, and wake rows chronological by `seq`), `diagnostics.json` (sparse per-event diagnostics from the closed code set in `/app/docs/diagnostics.md`, events sorted by `seq`, diagnostics within an event sorted by `(severity_rank, code, mutex)` with `null` mutex before any string), and `summary.json` (documented counters). Every output file must be canonical JSON: UTF-8, ASCII-only, two-space indent, lexicographically sorted keys at every depth, and a single trailing newline. The binary is rebuilt from `/app/environment` via the Makefile and run on `/app/data` on every invocation; prebuilt binaries or hand-written output JSON files are not acceptable submissions.
