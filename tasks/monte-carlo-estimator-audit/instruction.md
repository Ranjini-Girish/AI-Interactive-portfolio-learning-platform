A Java tool at `/app/src/mc/` must be completed (or rewritten) so that running `javac -d /app/build /app/src/mc/*.java && java -cp /app/build mc.Main` produces `/app/output/audit_report.json` — a JSON audit of four Monte Carlo integration methods applied to every test function in `/app/data/functions/` at every sample size listed in `/app/config/audit_params.json`.

Ten function-definition files live under `/app/data/functions/`, each a JSON object with fields `id`, `type`, `params`, `domain`, `exact_integral`, and `description`. Supported types and their evaluation rules: `polynomial` — `f(x) = sum_i coefficients[i] * x^i`; `trigonometric` — `f(x) = amplitude * sin(frequency * x + phase) + offset`; `exponential` — `f(x) = amplitude * exp(rate * x)`; `logarithmic` — `f(x) = amplitude * ln(scale * x + shift)`; `rational` — `f(x) = numerator / (x2_coeff * x^2 + x1_coeff * x + x0_coeff)`; `absolute_value` — `f(x) = |x - center|`; `gaussian_density` — `f(x) = amplitude * exp(-(x - mean)^2 / (2 * sigma^2))`; `step` — `f(x) = value_below if x < threshold, else value_above`; `oscillatory` — `f(x) = amplitude * cos(2 * periods * pi * x)`; `constant` — `f(x) = value`.

Pre-generated uniform [0,1] samples are in `/app/data/samples/samples_nNNNNN.json` (fields `sample_size`, `seed`, `values`). Method definitions are in `/app/config/methods.json` and audit parameters in `/app/config/audit_params.json`. Algorithm specifications are in `/app/docs/`.

For each (function, method, sample_size) triple, compute an `estimate` of the integral over the function's domain using the method described below, then derive `absolute_error = |estimate - exact_integral|`, `relative_error = absolute_error / |exact_integral|` (null when `exact_integral` is zero), `sample_variance` using population variance (denominator N) over the per-sample estimator values, `standard_error = sqrt(sample_variance / N)`, and `cost_adjusted_variance = sample_variance * evaluations_per_sample` (from the method config).

Crude MC: `estimate = (1/N) * sum f(x_i)`. Per-sample values are `v_i = f(x_i)`.

Antithetic Variates: pair each sample with its complement. Per-sample values are `v_i = (f(x_i) + f(1 - x_i)) / 2`. `estimate = (1/N) * sum v_i`. Variance is the population variance of the N values `v_i`, not of the 2N individual evaluations.

Stratified Sampling: partition [0,1] into K equal strata (K from `num_strata` in the method config). Transform within-stratum uniform draws: `x_{k,j} = (k + u_j) / K` for stratum k. Allocate the N samples across K strata: the first `N mod K` strata receive `ceil(N/K)` samples, the rest receive `floor(N/K)`, consuming samples from the input array sequentially. Compute per-stratum means, then `estimate = (1/K) * sum(stratum_means)`. The overall variance is `(1/K^2) * sum_k (sigma_k^2 / n_k)` where `sigma_k^2` is the within-stratum population variance (denominator `n_k`) and `n_k` is the stratum sample count. When a stratum has fewer than 2 samples its variance is 0.

Control Variates: control function is `g(x) = x` with known mean `E[g] = 0.5` (from `control_exact_mean` in the method config). Compute the optimal coefficient `c* = Cov(f, g) / Var(g)` where both covariance and variance use population formulas (denominator N). If `Var(g) = 0`, set `c* = 0`. Adjusted values: `v_i = f(x_i) - c* * (g(x_i) - E[g])`. Note that `E[g]` is the analytical value from the config, not the sample mean of g. `estimate = (1/N) * sum v_i`.

The `convergence` section reports the empirical convergence order between each consecutive pair of sample sizes for every (function, method): `order = log(error_small / error_large) / log(n_large / n_small)`. Report null when the error did not decrease (`error_small <= error_large`) or either error is exactly zero.

The `efficiency_summary` section, evaluated at the largest sample size only, reports for each function: `efficiency_ratios` mapping each method to `CAV_crude / CAV_method` (crude is always 1.0), null when either CAV is null or the method's CAV is zero; and `best_method`, the method with the highest efficiency ratio (ties broken by first in method list order). When all ratios are null, `best_method` is `"crude_mc"`.

Write the output to `/app/output/audit_report.json` as a single JSON object with four top-level keys. The `metadata` object contains the total function count, total method count, the list of sample sizes, and the seed read from the first sample file. The `results` array contains one object per (function, method, sample_size) triple, each carrying the function_id, method_id, sample_size, estimate, exact_integral, absolute_error, relative_error (null when exact integral is zero), sample_variance, standard_error, and cost_adjusted_variance; sort this array by method_id ascending, then function_id ascending, then sample_size ascending. The `convergence` array contains one object per consecutive sample-size pair per (function, method), each carrying function_id, method_id, n_small, n_large, error_small, error_large, and empirical_order (null when order is undefined); sort by method_id ascending, then function_id ascending, then n_small ascending. The `efficiency_summary` array contains one object per function, each carrying function_id, best_method, and an efficiency_ratios mapping from each method_id to its ratio (or null); sort by function_id ascending. All floating-point values rounded to 8 decimal places. Format with 2-space indentation and a trailing newline.
