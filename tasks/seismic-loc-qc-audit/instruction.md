Implement the seismic event localization and quality-control pipeline under `/app/`. The pipeline has four stages: (1) filter arrival picks for eligibility (excluded stations, min usable picks), (2) localize each processable event via a coarse-then-fine grid search against the velocity model, (3) estimate local magnitude from amplitude observations, and (4) emit QC findings (depth boundary alerts, azimuth gap warnings, magnitude outliers, station distance warnings, insufficient picks). Output is a single JSON report with four top-level keys: `schema_version`, `summary`, `events`, and `findings`.

A build skeleton is provided at `/app/seismic_audit.cpp` — extend it, replace it, or rewrite in Python; either works. Build and run via `make -C /app build && /app/bin/seismic_audit --data /app/data --out /app/output/localization_report.json`. The full algorithm specification, output schema, edge-case rules, and finding definitions are in `/app/docs/`. Don't modify files under `/app/data/`.
