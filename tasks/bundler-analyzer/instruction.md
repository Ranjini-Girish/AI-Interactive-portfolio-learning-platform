Write a TypeScript program that analyzes a set of module definitions and produces a bundle analysis report. Module definitions are in `/app/data/modules/*.json` and project configuration is in `/app/data/config.json`. Specification documents describing the module format, graph construction, tree-shaking algorithm, chunk assignment rules, and output schema are in `/app/data/module_spec.md`, `/app/data/graph_rules.md`, `/app/data/treeshake_rules.md`, `/app/data/chunking_rules.md`, and `/app/data/output_format.md`.

The program must build a dependency graph from static imports, detect all circular import chains, determine which modules and exports are reachable from the configured entry points, compute tree-shaken sizes, and assign modules to chunks (entry, async, and shared) based on dynamic import boundaries and the shared-size threshold. Namespace imports (`"*"`) mark all exports of the target as used. Modules with `side_effects: true` are kept in the bundle when reachable even if none of their exports are consumed, but their unused exports are still excluded from size totals. External packages are excluded from all graph and size analysis.

Compile the TypeScript source with `tsc` and run the compiled TypeScript with `TypeScript on Node 22`. Write the output to `/app/output/bundle_report.json` with two-space indentation and a trailing newline.
