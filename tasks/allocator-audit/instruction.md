Implement a memory allocator simulator in TypeScript on Node 22 that reads a pool configuration from `/app/data/pool_config.json` and twelve allocation traces from `/app/data/traces/`, simulates a first-fit allocator with block headers and alignment padding on each trace independently, and writes a combined audit report to `/app/output/allocator_report.json`. The compiled binary must be placed at `/app/build/allocator` and must be a native ELF executable (not a script wrapper). The environment provides `rustc` and `cargo` with `serde` and `serde_json` already available as dependencies in `/app/Cargo.toml`. Do not modify any files under `/app/data/`.

Each trace file contains a sequence of alloc, dealloc, and realloc operations on named blocks within a fixed-size memory pool. The allocator must account for per-block header overhead, alignment-based size rounding, immediate coalescing on deallocation, in-place realloc expansion when possible, and block splitting constraints. It must also detect and record errors such as double-frees, use-after-free, and out-of-memory conditions without halting. The detailed algorithm specification, alignment rules, coalescing policy, realloc behavior, error handling, metric formulas, and output schema are documented in the `.md` files under `/app/docs/`.

The output JSON uses 2-space indentation with a trailing newline. It contains pool_config, a traces array with per-trace results sorted by trace_id, and a summary with aggregate counts. Round all floating-point metrics to six decimal places.
