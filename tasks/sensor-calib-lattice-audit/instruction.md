The calibration team needs a deterministic audit of a sensor calibration workspace. Inputs live under `/app/sensor_lattice/`; the complete grammar, precedence rules, status values, duplicate incident handling, lineage rules, recall-window rules, and canonical JSON format are documented in `/app/sensor_lattice/SPEC.md`. Read that contract before producing output.

Write a Go program at `/app/src/main.go`, build an executable at `/app/bin/sensor-auditor`, and run it to emit exactly five UTF-8 JSON artifacts under `/app/calibration_audit/`: `calibration_plan.json`, `lineage_risk.json`, `recall_windows.json`, `lab_ledger.json`, and `summary.json`. The artifacts must reflect the current-day audit after sensor dependencies, batch contamination, recall extensions, lab freezes, suppressions, tier thresholds, lab bias, and lab capacity are applied.

When `SCLA_DATA_DIR` is set to a non-empty value it overrides the input root, and when `SCLA_AUDIT_DIR` is set to a non-empty value it overrides the output directory. Unset or empty values fall back to `/app/sensor_lattice/` and `/app/calibration_audit/`. Create the output directory if needed, and do not modify the input tree.
