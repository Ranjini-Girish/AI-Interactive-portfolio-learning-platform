Reliability telemetry needs a deterministic audit of relay hop byte budgets across integer epochs. Read the frozen dataset under `/app/relayhop/`, apply the staged incident list in file order at each epoch boundary, replay the flow cards with the admission rule given in the normative spec, and emit four UTF-8 JSON artefacts under `/app/audit/` named `admissions.json`, `denials.json`, `carry_ledgers.json`, and `summary.json`. Per-hop halt and resume handling, cumulative cap adjustments, bounded carry between epochs, and the all-or-nothing admission test for each flow are defined only in `/app/relayhop/SPEC.md`, which also defines every sort order and the canonical JSON layout for each output.

Each artefact must be canonical JSON with two-space indentation, ASCII-only content, sorted object keys, and a single trailing newline after the closing brace. When `RHC_DATA_DIR` is set it overrides the default read root `/app/relayhop/`; when `RHC_AUDIT_DIR` is set it overrides the default write root `/app/audit/`. Create `/app/audit/` when it is missing and never modify anything under the read-only data tree.
