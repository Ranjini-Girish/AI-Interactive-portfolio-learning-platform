Radiosonde sounding data from a six-station network is stored as individual JSON files under `/app/data/soundings/`, with station metadata in `/app/data/stations.json` and analysis parameters in `/app/data/config.json`. Write a TypeScript on Node 22 program that reads every sounding, computes a full thermodynamic profile analysis, and writes the results to `/app/output/analysis.json`.

Each sounding file contains an array of atmospheric levels with pressure (hPa), temperature (°C), dewpoint (°C), wind direction (degrees), wind speed (m/s), and geopotential height (m). Physical constants to use throughout: dry air gas constant R_d = 287.05 J/(kg·K), dry air specific heat c_pd = 1004.68 J/(kg·K), water vapor gas constant R_v = 461.52 J/(kg·K), ratio epsilon = R_d / R_v, standard gravity g = 9.80665 m/s², and reference pressure P_0 = 1000 hPa.

For every level in each sounding, compute saturation vapor pressure e_s via the Buck (1981) equation: e_s = 6.1121 × exp((18.678 − T_c / 234.5) × T_c / (257.14 + T_c)) where T_c is temperature in °C and e_s is in hPa. Actual vapor pressure e uses the same equation evaluated at the dewpoint. Mixing ratio is w = epsilon × e / (P − e) in kg/kg; note the denominator is total pressure minus vapor pressure, not total pressure alone. Relative humidity is 100 × e / e_s. Potential temperature is theta = T × (P_0 / P)^(R_d / c_pd) where T is in Kelvin. Virtual temperature uses the exact formulation T_v = T × (1 + w / epsilon) / (1 + w); do not substitute the linear approximation T × (1 + 0.608 w). Virtual potential temperature theta_v is computed identically to theta but substituting T_v for T.

Brunt-Väisälä frequency is reported as the signed frequency N, not N-squared. Compute N² = (g / theta_v) × d(theta_v) / dz using centered finite differences for interior levels and one-sided differences at the boundaries, then set N = sign(N²) × sqrt(|N²|). Negative N indicates statically unstable layers.

Lifted condensation level pressure and temperature are found using Bolton (1980): T_LCL = 1 / (1 / (T_d − 56) + ln(T_K / T_d) / 800) + 56, where T_K and T_d are the surface temperature and dewpoint in Kelvin. LCL pressure follows from the Poisson relation P_LCL = P_sfc × (T_LCL / T_K)^(c_pd / R_d).

CAPE and CIN are computed using virtual temperature. Lift the surface parcel dry-adiabatically (conserving theta) from the surface to the LCL, then pseudo-adiabatically above the LCL by integrating the moist-adiabatic temperature change in 1-hPa pressure steps, where the latent heat of vaporization varies as L_v = 2.501 × 10^6 − 2370 × (T − 273.15) J/kg. At each sounding level, compute the buoyancy b = g × (T_v,parcel − T_v,env) / T_v,env. The level of free convection is the first level above the LCL where the parcel virtual temperature exceeds the environment, and the equilibrium level is the first level above the LFC where the parcel is no longer warmer. CAPE integrates positive buoyancy between the LFC and EL using the trapezoidal rule over the sounding level heights. CIN sums negative buoyancy between the surface and LFC. If no LFC exists, CAPE is zero and CIN accumulates all negative buoyancy.

Precipitable water integrates mixing ratio over the sounding depth: PW = (1 / g) × sum of layer contributions, where each layer contributes (w_bottom + w_top) / 2 × |delta_P| with delta_P in Pascals, giving PW in mm.

Stability indices require values at standard pressure levels, which may not correspond to actual sounding levels. Interpolate temperature and dewpoint using log-linear interpolation in pressure: val(P) = val_1 + (val_2 − val_1) × ln(P / P_1) / ln(P_2 / P_1), where P_1 and P_2 bracket P. If a required standard level (850, 700, or 500 hPa) falls below the station's surface pressure, the corresponding index is null. K-index = (T_850 − T_500) + Td_850 − (T_700 − Td_700), all in °C. Total Totals = (T_850 + Td_850) − 2 × T_500. Lifted Index = T_500,env − T_500,parcel, where T_500,parcel comes from lifting the surface parcel to 500 hPa along the appropriate adiabat and converting back to °C.

Bulk wind shear over the 0–6 km AGL layer uses standard meteorological wind decomposition: u = −speed × sin(direction_rad), v = −speed × cos(direction_rad), linearly interpolating u and v components to 6 km above ground, then shear magnitude = sqrt(delta_u² + delta_v²).

The output JSON object has three top-level keys. Under "metadata", include the constants object and the total sounding_count. Under "profiles", an array of objects sorted by station_id ascending then launch_time ascending. Each profile contains station_id, launch_time, a levels array sorted by descending pressure, and a derived object. Level objects carry brunt_vaisala_Hz, e_hPa, e_sat_hPa, height_m, mixing_ratio_kgkg, pressure_hPa, relative_humidity_pct, temperature_K, theta_K, theta_v_K, virtual_temp_K, wind_direction_deg, and wind_speed_ms. The derived object holds bulk_shear_0_6km_ms, cape_J_per_kg, cin_J_per_kg, k_index, lcl_pressure_hPa, lcl_temperature_K, lifted_index, precipitable_water_mm, and total_totals. Null values appear for any index whose standard pressure level is below the surface. Under "summary", report max_cape_J_per_kg, the max_cape_station id, mean_precipitable_water_mm across all profiles, profiles_with_cape_count (CAPE > 0), and profiles_without_cape_count. Round thermodynamic temperatures and potentials to 4 decimal places, mixing ratios to 6, pressures and heights to 2, Brunt-Väisälä to 6, and CAPE, CIN, PW, indices, and shear to 2. Vapor pressures round to 4 decimal places. Serialize with 2-space indentation, keys sorted alphabetically within every object, and a single trailing newline.
