A multi-tenant backup pool's catalog and policy are spread across the JSON files under `/app/data/`: `pool_state.json`, `retention_policy.json`, `incident_log.json`, per-host `host_profiles/<host>.json`, and per-host `snapshots/<host>.json`. The complete contract — input validation rules, the tamper-containment propagation pass, retention semantics with per-host override merging, hash-chain integrity classification, the two-pass capacity-driven eviction (per-tier quota first, then global capacity), cascade eviction, and the exact schema of every required artifact — is documented in `/app/data/SPEC.md`; read that file before producing output. A Cargo project skeleton lives at `/app/auditor/` with a pre-warmed dependency cache, so `cargo build --release --offline --locked` runs without network access. Complete `src/main.rs`, build, and run `/app/auditor/target/release/auditor` to emit exactly five JSON files under `/app/audit/`: `retention.json`, `eviction_plan.json`, `integrity.json`, `host_summary.json`, and `summary.json`. The dataset under `/app/data/` is read-only and must not be modified.

All outputs must be UTF-8, indented with two spaces, with object keys emitted in sorted order at every level, and end in a single trailing newline; every list field must follow the sort order specified in `/app/data/SPEC.md`, so that two correct implementations of the same contract produce byte-identical files for the same input.
