Reliability needs a byte-accurate audit that replays timed arrivals against a shared per-window byte ceiling, applies staged ceiling bumps and shaves in log order, and records how remainder bytes move forward across window boundaries until a halt line converts overflow into drops instead of spill records. The policy file names the window width and shared ceiling, while the incidents file lists ordered mutations that bump or shave those ceilings plus a halt marker that changes how overflow is recorded.

Emit five UTF-8 JSON files under `/app/audit/` named `admissions.json`, `spills.json`, `drops.json`, `windows.json`, and `summary.json`. Canonical JSON layout, sort orders, incident ordering, and the split between spill carry and forced drops are defined only in `/app/poolspill/SPEC.md`. Inputs live under `/app/poolspill/`. When `PWS_DATA_DIR` is non-empty it overrides reads, and when `PWS_AUDIT_DIR` is non-empty it overrides writes, otherwise those defaults apply. Create the audit directory when missing and never modify anything under the data tree.
