Build a deterministic Java 21 tail-sampling decision engine for a synthetic distributed-trace span log. Read `/app/data/spans.json` (the flat span list, each with a 16-hex `trace_id`, an 8-hex `span_id`, a `parent_span_id` that is `null` for a root or names another span in the same trace, a `service`, an `operation`, an integer `start_unix_ms`, a non-negative integer `duration_ms`, a `status` from `{"ok","error","timeout"}`, and a string-keyed string-valued `attributes` map), `/app/data/policies.json` (an ordered policy list with `name`, `type` from `{"status_match","latency","attribute","service","probabilistic"}`, an `action` of `"keep"` or `"drop"`, and the type-specific match fields), and `/app/data/config.json` (with `min_spans_per_trace`, the trace-level `incomplete_action`, `orphan_action`, `cycle_action`, and `multi_root_action` each from `{"keep","drop"}`, an integer `now_unix_ms`, a non-negative `future_timestamp_threshold_ms`, and a `severity_ranks` object mapping `"error"`, `"warn"`, `"info"` to distinct integers).  Write five JSON files into the second-argv output directory (the verifier's primary run uses `/app/output/`): `sampling_decisions.json` (per `trace_id` the final `decision`, the `matched_policy` name or `null`, and the `reason` from the closed set in `/app/docs/output_format.md`, sorted by `trace_id` ascending), `policy_stats.json` (per policy `matched_count`, `kept_count`, `dropped_count`, and `type`, sorted by `name` ascending), `service_stats.json` (per service `span_count`, `trace_count`, `error_spans`, `timeout_spans`, `kept_traces`, `dropped_traces`, and `max_trace_duration_ms`, sorted by `service` ascending), `trace_diagnostics.json` (sparse entries drawn only from the closed code set in `/app/docs/diagnostics.md`, sorted as documented there), and `summary.json` (the documented totals and the `hottest_service` by `span_count`). Every output must be canonical JSON, defined as the byte-for-byte result of `json.dumps(obj, indent=2, sort_keys=True, ensure_ascii=True) + "\n"` (UTF-8, ASCII-only `\uXXXX` escapes, two-space indent, sorted keys at every depth, single trailing newline). The verifier compares the five output structures byte-for-byte against a Java 21 reference implementation that re-derives them from the same inputs, so do not hardcode answers and the Java 21 build must actually run.

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/tailsampler` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.
