Build a deterministic Java 21 tool that replays a synthetic shard-placement trace for a sharded data store. Read `/app/data/nodes.json` (the initial cluster, each node with `id`, positive integer `weight`, and `rack`), `/app/data/shards.json` (initial shards with `id`, `primary`, a sorted `replicas` list disjoint from `primary`, and a positive `size_bytes`), `/app/data/events.json` (a strictly increasing dense `seq` log of `node_join`, `node_leave`, `node_drain`, `node_resume`, `manual_move`, `rebalance_round`, and `shard_resize`), and `/app/data/policy.json` (`replication_factor`, `max_moves_per_round`, `rebalance_strategy="greedy_weight"`, and booleans `rack_awareness`, `auto_rebalance_after_join`, `auto_rebalance_after_leave`, `track_history`). json` key set) — lives in `/app/docs/`, one file per concept. JSON Schemas for every input and output sit in `/app/schemas/`, and `/app/examples/` carries minimal inputs plus expected outputs. The verifier treats those docs as the source of truth: diagnostics outside the documented code set, mis-ordered or mis-typed fields, deviations from the documented rebalance/drain/leave/manual_move semantics, or summary keys outside the documented set will all fail.

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/shardsim` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

Write exactly five JSON files into the second-argv output directory (the verifier's primary run uses `/app/output/`) (and no others): `cluster_state.json` (final node and shard layout, both sorted by `id`), `move_log.json` (every shard relocation in chronological order), `cluster_diagnostics.json` (per-event diagnostics drawn only from the closed code set documented under `/app/docs/diagnostics.md`, sparse and sorted by `seq`, then within an event by `(severity_rank, code, ref_id)`), `move_graph.json` (the node-to-node move-lineage DAG, populated only when `policy.track_history` is true, with every node id ever observed as a graph node and multi-vertex SCCs surfaced as cycles), and `summary.json` (the documented counters plus the sorted ASCII list of racks present at trace end). Every output must be canonical JSON: UTF-8, ASCII-only, two-space indent, lexicographically sorted object keys at every depth, and a single trailing newline.
