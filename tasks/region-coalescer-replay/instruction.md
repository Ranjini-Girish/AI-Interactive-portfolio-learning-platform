Build a deterministic TypeScript on Node 22 tool that replays a synthetic virtual-memory region map. Read `/app/data/regions.json` (the initial set of non-overlapping mapped regions, each with a string `id`, integer `base` and `size`, a `prot` from `{"r","rw","rx","rwx"}`, and an `owner`), `/app/data/events.json` (a strictly increasing dense `seq` log of `map`, `unmap`, `mprotect`, `split`, and `merge` operations), and `/app/data/policy.json` (with `coalesce_mode` in `{"strict","prot_match_only","prot_and_owner_match"}`, `overlap_action` in `{"reject","replace"}`, a positive `min_region_size`, `track_history`, and `auto_coalesce_after_unmap`).

Place `.ts` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Build with the project TypeScript toolchain into `/app/build/regionsim` and invoke that program with two positional arguments: input directory, then output directory. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

Write five JSON files into the second-argv output directory (the verifier's primary run uses `/app/output/`): `region_state.json` (the final non-overlapping region set, sorted by `(base, id)`), `coalesce_log.json` (chronological auto-coalesce events triggered by `map` or `unmap`, never by an explicit `merge`), `region_diagnostics.json` (per-event diagnostics drawn only from the closed code set documented under `/app/docs/diagnostics.md`, sorted within an event by `(severity_rank, code, region_id)`), `region_graph.json` (the lineage DAG built only when `policy.track_history` is true, with every region id ever observed as a node and edges from a parent region to each region produced from it by `split` or `merge`, deduplicated and sorted), and `summary.json` (the documented counters and the sorted ASCII list of owners still holding at least one region at trace end). Every output must be canonical JSON: UTF-8, ASCII-only, two-space indent, lexicographically sorted object keys at every depth, and a single trailing newline. The verifier reruns a reference implementation against the same `/app/data/` and compares the five structures byte-for-byte, so do not commit a prebuilt binary, do not hardcode answers, and the TypeScript on Node 22 build must actually run.
