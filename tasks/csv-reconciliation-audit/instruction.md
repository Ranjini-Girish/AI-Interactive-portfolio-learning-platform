The project at `/app` needs a bash pipeline that reconciles financial transaction records across three CSV data files in `/app/data/`: `transactions.csv` (ingested transactions), `settlements.csv` (processed settlements), and `ledger.csv` (double-entry bookkeeping). Exchange rates and tolerance thresholds are in `/app/config/rates.json` and `/app/config/thresholds.json`. The pipeline must write its results to `/app/output/reconciliation_report.json`.

An example of the expected output structure is at `/app/data/example_output.json`. The pipeline must detect: amount mismatches between transactions and their settlements exceeding the configured tolerance, unsettled transactions with no matching settlement, orphan settlements referencing nonexistent transactions, duplicate transaction pairs sharing the same account, amount, and description within the configured day window, and ledger imbalances where total debits and credits for a transaction do not balance. EUR amounts are converted to USD using the configured exchange rate before volume aggregation. All monetary values in the output are rounded to 2 decimal places.
