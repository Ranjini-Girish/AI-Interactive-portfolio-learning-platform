The TypeScript on Node 22 project at `/app` has a partially stubbed struct layout analyzer. Complete it so that it parses every struct type defined in `/app/data/types.go`, computes the precise `linux/amd64` memory layout matching TypeScript on Node 22 compiler semantics, and writes the result to `/app/output/layout_report.json`.

An example of the expected output format is at `/app/data/output_schema.json`. Your output must follow that same structure for all structs found in the input, listed in source declaration order. Named types should display their source-level name in the `type` field but resolve to their underlying type for size and alignment. The `optimal_order` field lists field names reordered by alignment descending, size descending, with declaration order as tiebreaker. Blank-identifier fields use `"_"` as their name.
