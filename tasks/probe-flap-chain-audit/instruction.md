Operations needs a frozen health-probe verdict for every node in the dependency forest after tier-scaled fail and flap thresholds, tier-specific rolling-window spans, rolling-window transition counts, post-fail soak windows, flap-day and fail-day suppressions, and incident-driven isolations and forced unhealthy directives are evaluated against the pool day in `/app/probeflaps/pool_state.json`. The result must be deterministic JSON that automation can consume without re-reading raw node files.

Produce five UTF-8 JSON documents under `/app/audit/` named `node_verdicts.json`, `tier_policy.json`, `incident_journal.json`, `dependency_touchpoints.json`, and `summary.json`. Every field name, sort order, status enum, and numeric rule tied to the pool day is defined in `/app/probeflaps/SPEC.md`, including the canonical JSON layout each file must use.

Bundled inputs live under `/app/probeflaps/`. When `PFCA_DATA_DIR` is set to a non-empty value it replaces the input root; when `PFCA_AUDIT_DIR` is set to a non-empty value it replaces the output directory. If either variable is unset or empty, reads default to `/app/probeflaps/` and writes to `/app/audit/`. Create the output directory when it is missing. The bundled `/app/probeflaps/` tree must not be modified by the audit run.
