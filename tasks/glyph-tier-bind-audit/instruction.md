A frozen glyph tier lab under `/app/gtb_lab/` ships policy JSON, a manifest whose tags may disagree, a current epoch counter, directed bind edges between sample ids, north and south lane biases, incident masks, ancillary witnesses, and numbered glyph traces. Read `/app/gtb_lab/SPEC.md` for stale-sample handling, manifest-driven fold stride, XOR-prefix histogram folding, cooldown echo increments, tier spill on non-stale samples only, and the bind-walk propagation pass that mutates upstream histograms after spill targets are recorded.

Write exactly `tier_bins.json`, `bind_events.json`, and `summary.json` into the audit directory using the canonical UTF-8 JSON rules in the specification: two-space indent, sorted keys at every depth, colon plus space separators, and one trailing newline per file. The summary must report stale sample counts, effective fold stride, bind propagation totals, and the tail epoch digest defined in the specification.

Read `/app/gtb_lab/` by default. Non-empty trimmed `GTB_DATA_DIR` replaces the lab root. Non-empty trimmed `GTB_AUDIT_DIR` selects the write directory, otherwise `/app/audit/`. Create the audit directory if missing. Never modify anything under the lab read root.
