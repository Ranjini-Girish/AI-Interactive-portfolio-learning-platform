Place Go sources under `/app/src/` with a `package main` entry point and compile them with Go 1.23 to `/app/bin/rwqaudit`. The read-only laboratory under `/app/rwq_lab/` lists segments on a closed ribbon, two anchor windows that add the same integer boost on every overlap, and day-scoped incident relaxations that lower the integer quorum for matching tiers while the evaluation day is still inside each window. Flux integers on the segments must close to zero under the modulus named in the policy file, or the ribbon is treated as broken before any per-tier capacity trimming runs.

Read `/app/rwq_lab/SPEC.md` for witness arithmetic, cyclic inclusive anchor walks, tier trimming order, diagnostic codes, and verdict strings. Emit exactly two UTF-8 JSON files into `/app/audit/` named `summary.json` and `segment_verdicts.json` using the canonical layout there (sorted object keys, no insignificant whitespace, and a single Unix newline after each file). Files under `/app/rwq_lab/ancillary/` are not inputs. When `RWQ_DATA_DIR` or `RWQ_AUDIT_DIR` is a non-empty string, use it instead of `/app/rwq_lab/` or `/app/audit/` respectively. Create the audit directory when it is missing.
