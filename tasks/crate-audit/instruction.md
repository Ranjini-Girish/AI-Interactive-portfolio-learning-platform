Implement a crate dependency resolver and auditor. The tool reads a package registry from `/app/data/registry/`, a project manifest from `/app/data/project.json`, and configuration from `/app/data/config.json`. It resolves all transitive dependencies using semantic versioning constraints, computes a topological build order, performs vulnerability and license audits, and writes a JSON report to `/app/output/audit_report.json`.

The registry contains one JSON file per package with version entries including dependency constraints, license identifiers, and vulnerability scores. The supported semver constraint operators, version comparison rules, resolution algorithm, vulnerability propagation model, license compatibility checking, and the complete output schema are documented in the `.md` files under `/app/docs/`. The tool must conform to every specification described there.

The output JSON uses 2-space indentation with a trailing newline. Float values are rounded to the precision specified in the config but trailing zeros after the decimal point may be omitted (e.g. `7.5` not `7.5000`). Place the compiled binary at `/app/build/crate-audit`. Do not modify any file under `/app/data/`, `/app/docs/`, or `/app/Cargo.toml`.
