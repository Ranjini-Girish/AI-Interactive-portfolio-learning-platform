Implement a workspace dependency health auditor in TypeScript on Node 22 that reads a workspace configuration from `/app/data/workspace.json`, member manifests from `/app/data/workspace/`, and a crate registry from `/app/data/registry/`, then resolves all dependencies, computes coupling metrics and health scores, audits license compatibility, and writes a comprehensive report to `/app/output/dep_health_report.json`. The environment provides `serde` and `serde_json` in `/app/Cargo.toml`; place the compiled binary at `/app/build/dep-audit`. Do not modify files under `/app/data/`.

The specification documents under `/app/docs/` are the authoritative reference for all computations. Read every file in `/app/docs/` carefully before implementing. The six spec files cover: semver parsing and range matching (`semver_spec.md`), dependency resolution and build ordering (`resolution_spec.md`), coupling metrics and version freshness (`metrics_spec.md`), license compatibility rules (`license_spec.md`), health scoring and grading (`health_spec.md`), and output structure (`output_format.md`).

All workspace members share a unified resolution: collect every version range for each crate across all members and their transitive dependencies, compute the intersection, and select the highest version within it. When no intersection exists the crate is a conflict — each member resolves it independently using its own constraints, and the crate is excluded from `unified_versions` and from the build order. Per-member `resolved_versions` must use the unified version for all non-conflicting crates.

The report must include coupling metrics, version freshness and staleness, per-member weighted staleness, license audit results, health scores with grades, a health ranking, a topological build order, and a summary — all computed exactly as specified in the docs. All floating-point values must be rounded to the precision specified in `workspace.json`. Output must use 2-space JSON indentation with a trailing newline, and all object keys must be sorted alphabetically.
