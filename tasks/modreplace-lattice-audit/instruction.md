An operator wants a single deterministic snapshot of the module replace lattice for a Go monorepo so downstream automation can act on a frozen view without replaying the raw manifests. The pool's manifests carry replace and require declarations whose effective resolution depends on scan order, calendar-based expiry, two prefix policies, per-module overlays, and pool-driven revoke and tail-strip passes; the auditor's job is to surface the final global replace winners, the per-module resolutions, the divergences between modules that share a require path, and the cycle structure of the union replace graph.

Inputs sit under read-only `/app/modgraph/` and include `SPEC.md`, `pool_state.json`, both files under `policy/`, `workspace/scan_order.json`, and one JSON file per module under `modules/`; never write under that tree. The authoritative field names, ordering rules, edge-case semantics, every `summary.json` counter, and the legacy strip, tail local-strip, pool-driven revoke, and expiry rules all live in `/app/modgraph/SPEC.md`. Emit exactly five UTF-8 JSON files under `/app/audit/` — `global_replace.json`, `resolution.json`, `skew_pairs.json`, `cycle_report.json`, and `summary.json` — each with two-space indentation, lexicographically sorted object keys at every depth, and exactly one trailing newline. Implement the auditor in Go as `package main` under `/app/src/`, build `/app/bin/lattice-auditor`, and run it with no arguments. Honor `MLA_MODGRAPH_DIR` and `MLA_AUDIT_DIR` overrides, defaulting to `/app/modgraph` and `/app/audit`.
