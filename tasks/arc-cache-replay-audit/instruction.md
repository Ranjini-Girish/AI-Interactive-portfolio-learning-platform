Build a Java 21 replay engine for a Weighted Adaptive Replacement Cache (W-ARC). The binary is `arcache`, invoked as `arcache <in_dir> <out_dir>`, arguments must be real directories (S_ISDIR, no symlinks/FIFOs). Sources in `/app/src`, `/app/include`; build `/app/build/arcache` (mtime strictly > every source). Read `events.json`, `config.json` from `<in_dir>`; write `cache_state.json`, `decisions.json`, `event_audit.json`, `summary.json`, `violations.json` into `<out_dir>` by creating regular file `<name>.partial` (observable mid-run) then `rename(2)`-ing each to `<name>` in alphabetical order. Normative references: `/app/docs/`, `/app/schemas/` (authoritative JSON Schemas), `/app/examples/`.

Place `.java` sources only under `/app/src/` (no `.cpp`, `.hpp`, or `.py` anywhere under `/app/`). At least 200 lines of authored source under `/app/src/`. Compile with `javac -encoding UTF-8 -cp /opt/gson.jar -d /app/build from /app/src/*.java and run the entry class with `java -cp /app/build:/opt/gson.jar <MainClass> <input_dir> <output_dir>`, or ship a runnable script at `/app/build/arcache` that accepts the same two positional arguments. Exit non-zero on wrong arg count, internal failure, or malformed input. During the run `/app/data/`, `/app/build/`, `/app/src/`, `/app/docs/`, `/app/schemas/`, and `/app/examples/` are immutable; writes belong only in the second-argv output directory.

W-ARC tracks four MRU->LRU lists T1, T2, B1, B2 over cache size `c` in `[1, 256]` with `p` in `[0, c]` initial 0. Each resident has `cum_weight` initialised to the triggering access weight and incremented by it per hit; each ghost has `entry_weight` snapshot of `cum_weight` at demotion. `REPLACE` demotes the **min-`cum_weight`** entry of the side chosen by the canonical ARC predicate, LRU as tiebreak. The miss-case T1 direct-drop uses the same min-`cum_weight`+LRU rule; B1/B2 ghost drops are plain LRU. Ghost-hit deltas, access cases, evict/clear semantics, and population invariants follow canonical ARC.

Payloads: `access`=`{key, weight}` with `weight` in `[1, 16]`; `evict`=`{key}`; `clear`=`{}`. Output shapes: `cache_state.json`=`{p, t1, t2, b1, b2}`, T1/T2 entries `{cum_weight, key}`, B1/B2 entries `{entry_weight, key}`; `decisions.json`=`{decisions:[...]}`; `event_audit.json`=`{events:[...]}` sorted by `event_id`; `violations.json`=`{violations:[...]}` equals the rejected subset of `event_audit.events` sorted by `event_id`; `summary.json` is flat. Enum values (`outcome`, `reason_ignored`, `replaced_from`, `dropped_from`) are pinned in `/app/schemas/`. Every file is byte-exact under `json.dumps(obj, indent=2, sort_keys=True, ensure_ascii=True) + "\n"`. Refuse to start if any final or `.partial` sibling exists in `<out_dir>`; on success the only new entries are exactly the five final outputs. Exit non-zero on wrong argument count, missing/non-real-directory paths, malformed JSON, schema violation (extra/missing keys, wrong types, out-of-range integers, duplicate `event_id`), or IO failure; on rejection introduce no new entry into `<out_dir>` and leave every pre-existing entry byte-identical. Identical inputs produce byte-identical outputs.
